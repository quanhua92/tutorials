Speculative Decoding — interactive .md .py

The memory wall, fixed: a small draft model guesses K tokens ahead; the big target model verifies all K in ONE forward pass. A rejection-sampling trick makes the output exactly match the target distribution. Every number is recomputed in JS from the identical formula in speculative_decoding.py.  |  sampling (rejection math) · kv_cache (rewind on reject) · causal_mask (parallel verify) · scheduler (batch spec rounds)

decode: 1 FLOP/byte (memory-bound, GPU starved) spec verify: K FLOP/byte (amortized weight load) GPU peak: ~208 FLOP/byte (A100 FP16)
What to do: drag K and watch the spec bar grow. Decode stays at 1 FLOP/byte (0.5% of peak). Spec verify scales linearly with K — each extra draft token amortizes one more weight load. The GPU peak line shows how far we still are from compute-bound.

③ Acceptance ratio min(1, p/q) — why some tokens always pass

q(x) draft prob p(x) target prob min(1,p/q) accept ratio
What to do: tokens where p ≥ q (the & on) always pass (ratio=1). Tokens where p<q (cat, sat) pass with probability p/q. The adjusted distribution P_adj only puts mass on tokens where p>q.
