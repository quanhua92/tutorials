Separate Chaining Hash Table — interactive 📖 CHAINING.md 📄 chaining.py

📖 Full guide in the .md file  ·  📄 Ground-truth source code in the .py file.
Each bucket holds a linked list (a chain). A collision just prepends a node — the table never fills up. Performance is set by one knob, the load factor α = n/m: search is O(1 + α). Slide keys in, watch chains grow and the resize fire, then see why chaining degrades gracefully while open addressing blows up near α=1. Every number is recomputed in JS from the identical formulas as chaining.py, gold-checked below.

① Build the table — 8 buckets, 12 keys (α = 1.5), chains grow on collision

What to do: drag keys up to 12 and watch each key hash to its bucket h(k)=k mod 8 and prepend to the chain. Collisions lengthen a chain; empty buckets stay empty. The α gauge tracks fullness.
h(k) = k mod 8  ·  no resize (Section A)

② Search — hash to the bucket, then walk the chain key by key

What to do: pick a key. The page hashes it, then highlights each node walked (a pointer follow = one comparison). Present keys stop when found; absent keys walk the entire chain. Worst case = the longest chain (bucket 3, length 3).
m = 8, full table (α = 1.5)  ·  Section B
Avg successful search ≈ 1.50 comparisons (theory 1+α/2 = 1.75); avg unsuccessful ≈ 1.42 (theory α = 1.5). Search is Θ(1+α) — CLRS Thm 11.2.  © chaining.py Section B

③ Resize + rehash — when α > 0.75, double m and rehash every key

What to do: step inserts. At key #7, (6+1)/8 = 0.875 > 0.75 so m doubles 8→16 and all 6 keys are rehashed (each key's bucket is recomputed as k mod 16) before the new key is added. α drops from 0.875 to 0.4375.
threshold = 0.75, double on resize  ·  Section C
The rehash is O(n) on the day it fires but rare, so insert is O(1) amortized (same geometric-series argument as array doubling). Collisions still occur after rehash (bucket 11 keeps a chain of 3) — rehashing redistributes, never guarantees zero collisions.  © chaining.py Section C

④ Chaining vs open addressing + the balls-into-bins math

Left: expected unsuccessful-search probes vs α. Chaining is a gentle line 1+α; open addressing is 1/(1−α) with a vertical asymptote at α=1. Right: the exact chain-length distribution for one bucket (Binomial(12, 1/8)) — long chains are possible but exponentially rare.
