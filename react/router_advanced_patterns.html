three gates around every navigation

A route is more than a component. The router runs a lifecycle around each navigation: beforeLoad can throw redirect() (the auth gate), a blocker can pause or cancel the transition (the unsaved-changes gate), and masking decides which URL lands in history vs. which route actually renders. The order matters: block check → auth check → commit URL → render.

patterngateAPIwhat it can do
Authenticated route before navigation commits beforeLoad + throw redirect() abort → send to /login, remember the target, redirect back after auth
Navigation blocking before navigation commits useBlocker({ shouldBlockFn, withResolver }) pause → show "unsaved changes?" UI → proceed() or reset()
Location masking at commit (history write) <Link mask={{ to }}> / createRouteMask() render route A, but persist a clean URL B to history + the address bar
