📖 Read the full guide — the free-sampling → regex/DFA → GBNF lineage, the −∞-mask identity, the worked decode, killer gotchas, cheat sheet. This page is the interactive companion.
The bouncer idea: the model scores every token; the grammar mask tears up the tickets of tokens that would break the schema (logit → −∞), so softmax can only land on a valid token. Toggle the mask, step a decode, and watch validity hit 100%. Every number is recomputed in JS from the identical formula in grammar_masking.py.  |  compare with RAG slim →

① The mask in action: one logits vector (the GOLD anchor)

How to read: the toy model's TOP preference is the letter 'a' (logit 5.0) — but a JSON number can never start with a letter. Toggle the mask: ON sets every disallowed logit to −∞ (grey) and softmax renormalizes over the allowed-set only, so '-' wins. The GOLD pin is the masked P('-') = 0.213730 (matches grammar_masking.py Section B).

② Autoregressive decode viewer — step through the mask, build '-12.5'

How to read: each step the model "wants" an invalid char (its free top, shown red). The mask recomputes the allowed-set from the current DFA state and picks the best valid char (teal). STOP is only allowed when the state is accepting — so the decode can only end on a complete number. Drag the step slider.
0 = start · 6 = final (STOP)

③ Reliability: mask ON vs OFF over N toy decodes

How to read: for each of N trials a different (seeded) logits schedule; greedy-decode once masked, once free; test with the regex. Masked is ALWAYS 100% (validity is guaranteed by construction, not measured). Free sampling mostly emits letters / stops early / malformed numbers.
