📖 Read the full guide — architecture diagrams, decision tables, killer gotchas, cheat sheet. This page is the interactive companion.
An online judge = a priority submission queue + sandboxed code execution (gVisor/Firecracker + cgroup limits) + a test-case runner (compile → run → compare). Watch a submission flow through the pipeline, adjust the cgroup resource limits to see verdicts change, step through the test runner, then explore the contest leaderboard — all recomputed in pure JS and gold-checked against coding_platform.py.

① Submission lifecycle — watch code flow through the judge pipeline

What to do: click Submit (contest) or Submit (practice) to animate a submission through the 7-stage pipeline. The contest lane (priority 0) always dequeues before practice (priority 1) — see the queue state below (coding_platform.py Section 1).
Click a submit button to start. Each stage takes ~400ms to visualize the flow.

Queue state — language partitions, priority lanes

Demo: 6 submissions across python/cpp/java. Contest (gold) always pulls before practice (grey), FIFO within each lane. Matches .py Section 1 exactly.

② Sandbox resource limiter — drag cgroup limits, watch the verdict flip

What to do: pick a program behavior and drag the memory.max / cpu.max sliders. The resource bars show usage vs the limit; the gold tick marks the cgroup ceiling. Raise memory.max above 300MB and the hungry program flips from MLE to AC (.py Section 2).
—

③ Test runner flow — compile → run each test → compare → verdict

What to do: pick a program and Run all tests. The judge compiles (or hits CE), then runs each Two Sum test case under the cgroup limits, stopping on the first failure. Verdict precedence: CE > TLE > MLE > RE > WA > AC (.py Section 3).
Click “Run all tests” to start the judge pipeline.

④ Contest leaderboard — composite-score sorted set (solved × 1e9 − penalty)

What to do: the standings are ranked by solved DESC, penalty ASC. Penalty = accept_time + 5min per wrong submission (only if the problem is eventually accepted). The composite score packs both keys into one float so ZREVRANGE sorts in one pass. Alice beats carol on the 3-solved tie because her penalty (30min) < carol's (75min) (.py Section 5).
Scores: solved × 1,000,000,000 − penalty_seconds. Penalty in parentheses shown in minutes.

⑤ Scale calculator — sweep inputs, watch storage / QPS / sandbox fleet move

What to do: drag the sliders. Defaults reproduce coding_platform.py Section 6 exactly (2M subs/day, 100 tests/problem, 3s/sub).
