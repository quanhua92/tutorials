LZW (Lempel–Ziv–Welch 1984) — interactive LZW.md lzw.py

A dictionary coder that coins new abbreviations as it reads. The dictionary grows itself one entry per emitted code — and the decoder rebuilds the exact same dictionary from the code stream alone, so it is never sent. ② The one subtle corner — decoding 'AAAA' (entry = w + w[0])

Input 'AAAA' encodes to codes [1, 2, 1]. The encoder coins code 2 (AA) and immediately uses it on the next step — so the decoder receives code 2 before it has built that entry. The fix: the missing entry is always w + w[0], because a brand-new code only ever extends the current run by its own first char.
Step 2 is the special case: code 2 is not in the decoder's table yet, so it manufactures the entry as w + w[0] = 'A' + 'A' = 'AA'. Get this wrong and decode silently corrupts.

③ Compression ratio — fixed 12-bit vs variable width

ratio = total_code_bits / (chars × 8). Variable width (codes grow 1 bit at each 2k boundary, up to 12) always beats fixed-12-bit, because early codes ride at 2–3 bits instead of 12. That adaptive width is where LZW's compression actually comes from.
