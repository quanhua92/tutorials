② Inverted index explorer — click a term to walk its posting list

What to do: click any term. The posting list is the set of docs whose body contains it and the term frequency. Retrieval = merge a few posting lists (O(matches), never O(corpus)) — why BM25 over 100M docs is <25ms (search_ranking.py Section 1).

③ Query expansion — dictionary + pseudo-relevance feedback

What to do: a literal query misses semantically related docs. Dictionary synonyms (learning → deep, neural, training) plus Rocchio PRF (top docs' weighted terms) broaden it. Watch d3 (“deep learning neural network training”) jump from rank 4 → rank 2 (search_ranking.py Section 4).

④ Learning-to-rank — drag feature weights, watch NDCG move

What to do: BM25 ranked short off-topic d6 above longer on-topic d3. LTR folds in clicks/CTR/freshness/title-match and flips them. Drag the weights (they auto-renormalize) — when the order reaches d1,d2,d3,d6, NDCG@4 = 1.0000 (vs BM25's 0.9854). Defaults reproduce search_ranking.py Section 5.
