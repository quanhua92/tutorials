① Creation timeline — control-plane admits, scheduler binds, kubelet runs

What to do: add/remove init containers and stretch the main container's run. Watch the component column hand off from the control plane (kubectl → API server → scheduler) to the data plane (kubelet → container runtime). The phase never leaves Pending until every init container exits 0.
slider below = simulation clock
Read it as four acts: (1) admit to etcd, (2) scheduler binds a Node, (3) init containers run in order while the Pod is still Pending, (4) main container starts → phase flips to Running.

③ CrashLoopBackOff — exponential back-off, capped at 300s

What to do: drag the number of restarts. The kubelet delay min(10 × 2n, 300) doubles each crash. The wall-clock bar shows how fast a tight crash loop bogs down to a 5-minute cadence — that long gap between 'Running' flickers is the visible signature.

④ Graceful shutdown — preStop → SIGTERM → grace → SIGKILL

What to do: drag preStop duration and the app's drain time. preStop runs before SIGTERM and its time counts against the grace period. If preStop + drain > grace → the kubelet SIGKILLs (unclean, in-flight requests dropped).

⑤ QoS classes — computed from requests vs limits

What to do: toggle each container's CPU & memory request/limit. Guaranteed = every container has cpu+mem request==limit (all set); BestEffort = nothing set anywhere; anything in between is Burstable. Eviction order under memory pressure: BestEffort → Burstable → Guaranteed.
