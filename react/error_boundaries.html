the catch net: why error boundaries exist

When a component throws during render, React unmounts the entire component tree — the whole screen goes white. An error boundary is a class component that acts as a try/catch for its subtree: it catches the error and renders a fallback UI instead of crashing the app. Define static getDerivedStateFromError(error) to update state (pure — return the new state), and componentDidCatch(error, info) for side effects (logging, analytics).

async code (fetch, await, promises) NO .catch() / try / catch
setTimeout, setInterval, requestAnimationFrame NO try / catch inside the callback
errors thrown IN the boundary itself NO a parent boundary, or it crashes
