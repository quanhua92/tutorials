Run-Length Encoding (RLE) — interactive RLE.md rle.py

RLE replaces each maximal run of identical symbols with one (count, symbol) token. It is the simplest compressor there is — and it ONLY works when the data has long runs. Type below and watch the encode step-by-step, the compression ratio, and the one failure mode (alternating data expands 2×).  |  Huffman codes symbols by frequency →

② Compression ratio — RLE only wins when the average run > 2

What to do: pick a pattern and watch the ratio bar. The math is ratio = 2 / average_run (one token = 2 bytes, one symbol = 1 byte). Breakeven is avg run = 2; below that, RLE expands.

bits out (16/token)
The bars are drawn on the same scale. When orange (out) is shorter than blue (in), RLE compressed. When orange is taller, RLE expanded the data — the #1 reason RLE is never used alone on text.
