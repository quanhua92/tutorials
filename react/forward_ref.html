ref is special: forwardRef unwraps it

ref is not a normal prop — React intercepts it before your component ever sees it (in React 18 and earlier, function components silently drop it). React.forwardRef wraps your component so the parent's ref arrives as the second argument: (props, ref) => {'<'}element ref={ref}/{'>'}. useImperativeHandle then narrows what that ref exposes — instead of the raw DOM node, you return a custom API object. The parent calls inputRef.current.focus() and never knows there's an <input> underneath.

The parent holds inputRef and calls into the child's custom API. The gold-check exercises every method automatically: focus() (asserts the input is document.activeElement), sets the DOM value + getValue() (asserts the parent read it), then clear() (asserts the input was emptied). All four prove the child exposed a deliberate ref API — not its DOM node.
