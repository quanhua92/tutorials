① Live leaderboard — ZADD a score, watch it re-sort

What to do: type a player + score and submit. The dual-structure sorted set (skip list + hash table) re-ranks in O(log N). Toggle GT to enforce monotone-best (a regression score is rejected). Note the tie-break: equal scores are ordered by member descending — that is Redis' rule (gaming_leaderboard.py Section 1).

② Top-K pagination — ZREVRANGE offset offset+k-1

What to do: page through the demo set. Each page is a single ZREVRANGE start stop seek — O(log N + K) regardless of how deep you page. In production this is served from a 1s-TTL top-K cache; the ZSET is only hit on a miss (Section 3).

③ Tier buckets — 10,000-player deterministic sim

What to do: the bars are a deterministic 10K-player sim (score(i) = (i * 2654435761) % 100000). Drag the platinum threshold to re-bucket — lower thresholds swell the elite tier. Tier buckets are themselves segmented ZSETs, so bronze #1 and platinum #1 are both “first place” (gaming_leaderboard.py Section 4).
