Quickselect (Hoare 1961) — interactive QUICKSELECT.md quickselect.py

Quicksort that only walks ONE branch. Partition around a pivot, then recurse into only the side that holds the target index k — the orange bar marks the shrinking search region; everything outside it is thrown away. Average O(n) (vs O(n log n) to fully sort). The catch is the same as quicksort: a bad pivot degrades to O(n²). Watch the median-of-medians fix on hostile input. The worked trace is recomputed in JS from the identical array as quickselect.py.  |  reservoir_sampling: one-pass sampling →

① The algorithm — partition, then walk ONE side toward index k

What to do: press play (or step). Each level partitions the search region around a pivot; the pivot lands in its final position. If that position is k, we're done. Otherwise we discard the half that can't contain k and recurse into the other half only — that single-sided recursion is what makes this O(n) instead of O(n log n).
in search region ≤ pivot (small side) pivot scanning j (≤ pivot) / locked scanning j (> pivot) discarded (outside region) search region [lo..hi]

② Why one branch is O(n) — quickselect vs full sort

What to do: drag n. Quickselect does n + n/2 + n/4 + … = 2n comparisons (linear), because each level shrinks the region ~in half. A full sort does n log₂n. Quickselect's fraction of the sort cost is ~1/log₂n — vanishing as n grows.
random input, median-of-3 pivot (deterministic)
In words: need ONE order statistic (median, P95, top-k)? Select in O(n). Need the whole array in order? Sort in O(n log n). The gap is exactly log₂n.

③ The O(n) guarantee — median-of-medians on hostile input

What to do: drag n. On sorted/reversed input a plain last pivot peels one element per level → O(n²) (red). median-of-medians (BFPRT) spends extra work to pick a pivot in the 30–70% range, so it stays linear on every input. This is the fallback inside std::nth_element (introselect).
grey line = n (linear); red zone = O(n²) blowup
In words: quickselect's O(n²) hole (🔗 quicksort has the identical hole) is exactly what median-of-medians closes — same role heapsort plays in introsort. The trade is a ~5–10× higher constant, so real libs use quickselect and only fall back to mm past a depth limit.
