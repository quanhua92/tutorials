③ Throughput & memory calculator — per-stage end-to-end

What this shows: drag sources / rate / line size and see events/s, MB/s, RAM, and CPU per stage. The O2 Ingester ceiling is 31 MB/s single-node; past it the pipeline saturates (backpressure). Mirrors openobserve_ingest.py Section H gold-check.
FB parse ~2us/event, VRL ~9us/event. O2 Memtable 256MB, FB buffer 10MB, Vector batch ~0.5MB in-flight.

④ Schema evolution visualizer — new fields auto-added, no reindex

What this shows: as instrumentation ships new fields over days, O2 auto-detects each on ingest and adds a Parquet column. Old files simply lack the column (NULL). Click a day to see the schema grow; bar height = storage cost (low vs high cardinality). Mirrors openobserve_ingest.py Section F.
Parquet is columnar: a low-cardinality column (env, level) adds ~3KB/file; high-cardinality (user_id, trace_id) adds ~800KB/file. No reindex ever.
