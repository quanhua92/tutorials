① Dashboard JSON builder — pick a panel, see the model

What this shows: a dashboard is one JSON object. Pick a panel type, a data source, and a query; the panel's slice of the model (gridPos, datasource, targets) is generated live. The same JSON is what you export, import, and commit.
gridPos is on a 24-column grid. datasource.uid can be a $variable so the dashboard is source-agnostic across envs.

② Rendering cost — panels × refresh × viewers

What this shows: every refresh fires P×Q queries against your backends; aggregate load scales with concurrent viewers. A 24-panel wall dashboard at 5s with 100 war-room onlookers = 480 queries/sec. The refresh slider is a cost dial.
Levers (in order): 1) slow the refresh — the biggest dial. 2) query recording-rule outputs (offload to Prometheus). 3) shared queries. 4) fewer panels. 5) caching.

③ Alerting — states, for: gate & policy routing

What this shows: a rule goes Normal→Pending→Alerting, gated by for: duration. Then firing instances route through a label-matched notification policy tree (first-match-wins) to a contact point. Adjust the threshold / for: and watch the fire time move.

④ Provisioning — datasources & dashboards as code

What this shows: the YAML Grafana loads at startup. Toggle datasources / options; secrets stay in secureJsonData. A file provider polls the dashboard dir every 30s — a merged PR is live with zero UI clicks.
