# Architecture Visualizers

Interactive HTML visualizers for LLM architectures. Generated by `scripts/generate_viz.py` from HuggingFace `config.json` (no weights needed).

## Usage

```bash
# Generate for any model
python scripts/generate_viz.py Qwen/Qwen3-0.6B
python scripts/generate_viz.py google/gemma-4-E2B-it
python scripts/generate_viz.py tencent/Hy3 --subfolder checkpoint_folder_textonly
```

## Hand-Crafted Reference

`qwen3_0_6b_architecture_visualizer.html` is the original hand-crafted visualizer for Qwen3-0.6B, built by fact-checking every value against the HuggingFace config and source code. It has richer subgraphs than the generated files:

- RMSNorm shows the full 8-step formula (x-square, mean, +eps, rsqrt, scale, weight)
- RoPE shows the rotation matrix and frequency computation
- QK-Norm explains the per-head normalization and why it's applied before RoPE
- Every projection shows exact weight shapes and the "no bias" note
- Embedding lookup explains weight tying (lm_head = embed_tokens)
- Verified against `huggingface.co/Qwen/Qwen3-0.6B/raw/main/config.json` and HF source code

All other `_viz.html` files are auto-generated by `generate_viz.py` from config.json alone. They share the same template (`viz_template.html`) with values substituted per-model. The hand-crafted version serves as the quality reference for what the generator aims to approximate.

## How to Use

### View the dashboard

Open `index.html` in a browser. The split-view shows a searchable model list on the left and the selected visualizer in an iframe on the right.

- **Desktop** (>=768px): sidebar is always visible
- **Mobile** (<768px): tap `☰ Models` to open the sidebar as an overlay drawer; selecting a model auto-closes it
- **Search**: type in the search box to filter by model name or architecture type
- **Filters**: click Qwen / Google / DeepSeek / NVIDIA / Microsoft / Other to narrow by vendor
- **Keyboard**: press Up/Down arrows to cycle through models
- **Deep link**: append `#gemma-4-e2b` to the URL to open a specific model directly

Each visualizer is a standalone interactive HTML file — pan with drag, zoom with scroll, click any node to drill into a subgraph.

### Add a new model

**Step 1: Generate the visualizer**

```bash
cd ~/workspace/tutorials/llm-architectures

# Basic: fetches config.json from HuggingFace, generates <name>_viz.html
python generate_viz.py Qwen/Qwen3-8B

# With custom output name
python generate_viz.py google/gemma-4-E2B-it -o gemma4_e2b.html

# For manifest-style repos (Nemotron Audex, etc.)
python generate_viz.py nvidia/Nemotron-Labs-Audex-2B --subfolder checkpoint_folder_textonly
```

The script only fetches `config.json` (a few KB) — no model weights are downloaded. It auto-detects:
- QK-Norm (Qwen3, Gemma2/3/4, Cohere)
- Attention type (GQA / MQA / MHA)
- MLP type (SwiGLU / GeGLU / plain MLP with ReLU/ReLU-squared)
- RoPE theta (including nested `rope_parameters` and dual RoPE in Gemma 4)
- MoE experts count
- Weight tying, sliding window, logit softcapping
- Nested `text_config` (Gemma 4, VL models with vision/audio configs)

**Step 2: Register it in the dashboard**

Open `index.html` and find the `MODELS` array (near the bottom, inside `<script>`). Add one line:

```javascript
var MODELS = [
  // ... existing entries ...
  {f:"Qwen_Qwen3-8B_viz.html",n:"Qwen3-8B",t:"qwen3",p:"8.2B",a:"GQA 4:1",q:"Y",m:"SwiGLU",g:"qwen",e:""},
];
```

| Field | Description | Example |
|---|---|---|
| `f` | HTML filename | `"Qwen_Qwen3-8B_viz.html"` |
| `n` | Display name | `"Qwen3-8B"` |
| `t` | Architecture type (from config `model_type`) | `"qwen3"` |
| `p` | Param count (short form) | `"8.2B"` |
| `a` | Attention type | `"GQA 4:1"`, `"MQA"`, or `"MHA"` |
| `q` | QK-Norm? `"Y"` or `"-"` | `"Y"` |
| `m` | MLP type | `"SwiGLU"`, `"GeGLU"`, `"MLP"` |
| `g` | Vendor group for filtering | `"qwen"`, `"google"`, `"deepseek"`, `"nvidia"`, `"microsoft"`, `"other"` |
| `e` | MoE badge (empty string if not MoE) | `"MoE"` or `""` |

The model count badge in the top bar updates automatically based on visible items.

### Add support for a new architecture type

If `generate_viz.py` encounters an unknown `model_type`, it still generates a valid visualizer — it just won't detect architecture-specific features (QK-Norm, activation, etc.).

To add detection for a new type, edit `generate_viz.py`:

```python
# 1. Add to QK_NORM_TYPES if the architecture uses QK-Norm
QK_NORM_TYPES = {..., "new_type"}

# 2. Add to ACTIVATION_MAP if it uses a non-standard activation
ACTIVATION_MAP = {..., "new_act": ("NewGLU", "NewAct")}

# 3. Add to NO_ROPE_TYPES if it doesn't use RoPE
NO_ROPE_TYPES = {..., "new_type"}
```

### How the template works

`viz_template.html` contains a single HTML page with JavaScript that reads a `CFG` JSON object (injected by `generate_viz.py`) and builds the entire graph dynamically:

- `buildNodes(CFG)` — creates the main graph nodes, conditionally adding QK-Norm / RoPE nodes based on config
- `buildEdges(layout)` — wires nodes with correct tensor shapes and routing (v_proj bypasses QK-Norm/RoPE)
- `getSubgraph(nodeId)` — returns a detailed subgraph for each clickable node (RMSNorm formula, RoPE rotation, GQA attention, SwiGLU gate, etc.)
- Rendering is shared across all models — only the `CFG` object changes

The template uses `__CFG__`, `__SPECS__`, `__TITLE__`, `__MODEL_NAME__` as placeholders that `generate_viz.py` replaces.
