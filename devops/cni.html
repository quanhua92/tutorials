Each node owns a pod CIDR (/24); a pod IP's third octet names its node. The plugin decides how a packet crosses node boundaries: Calico routes directly over BGP (pod IPs visible), Flannel wraps the frame in VXLAN (pod IPs hidden), Cilium routes directly but in eBPF (skips iptables). ① Pod CIDR allocation — each node owns a /24, third octet names the node

What to look at: the cluster pod network 10.244.0.0/16 is carved into per-node /24 pod CIDRs. A pod draws its IP from its node's slice, so 10.244.3.7 lives on node-3. The host-local IPAM plugin leases each IP as a file so it never double-allocates.

③ Network policy — default allow vs default deny + label rules

What to do: toggle the db-allow-frontend policy on. With NO policy, everything reaches db-0 (default ALLOW). Once the policy selects db-0, it flips to default-DENY and only tier=frontend sources get through — db → db is blocked.
