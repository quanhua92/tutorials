Experiment Design — interactive EXPERIMENT_DESIGN.md experiment_design.py

Design and read a valid A/B test end-to-end: power a sample size from baseline + MDE, read the p-value & confidence interval off observed results, apply Bonferroni across the metric family, and check Sample Ratio Mismatch before trusting anything. Every formula recomputes live in JavaScript from the identical logic as experiment_design.py, gold-checked at the bottom.

① Sample size calculator — power analysis from baseline + MDE

What to do: set the baseline conversion rate, the minimum detectable effect (MDE), and the desired power. The per-arm sample size comes from the two-proportion z-test formula (experiment_design.py Section 1). Defaults reproduce the DoorDash-style scenario: 8% baseline, +0.5pp MDE, 80% power → 47,525 per arm.

② Significance checker — observed results → p-value + verdict

What to do: type in the control and treatment conversions and sample sizes. The two-sided two-proportion z-test runs live (experiment_design.py Section 2). Defaults reproduce control 5,400/50,000 vs treatment 5,750/50,000 → z 3.5164, p 0.000437, SIGNIFICANT.

③ Confidence interval — the treatment-effect band, visualized

Tracks the checker above. The 95% Wald CI for p_treatment − p_control (unpooled SE). If the band excludes zero the effect is significant; if its lower bound clears the MDE, it is also practically shippable (experiment_design.py Section 3).

④ Multiple testing — Bonferroni across the metric family

What to do: the metric family is 1 primary + 4 guardrails/secondary (k=5). With k tests at α=0.05 you expect ~0.25 false positives by construction. Bonferroni divides α by k (=0.01) and controls the family-wise error (experiment_design.py Section 4). Toggle the correction to watch metrics drop out.
k = 5 tests  ·  family-wise error @0.05 ≈ 0.23

⑤ Effect size — does it MATTER, not just is it REAL?

Statistical significance scales with n; effect size does not. Cohen's h (proportions) and Cohen's d (continuous) quantify magnitude independently of sample size (experiment_design.py Section 5). The h band reads off the checker in Panel 2.

⑥ SRM — Sample Ratio Mismatch (the silent killer)

What to do: edit the observed assignment counts (intended 50/50). A chi-square goodness-of-fit (df=1) tests the split. p < 0.001 → HALT, do not rebalance (experiment_design.py Section 6). Defaults: healthy (50,050 / 49,950) vs mismatch (52,000 / 48,000).
