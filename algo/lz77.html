A sliding window = a search buffer (the dictionary) + a lookahead. At each step the encoder finds the longest run in the search buffer that matches the lookahead's start, then emits a (distance, length, next-char) back-reference instead of the bytes. ② The window-size sweet spot (longer text, L = 15)

What to do: the phrase 'the quick brown fox…' repeats ~45 chars apart. Drag W below 45 and the window can no longer reach the repeat, so the encoder emits mostly literals and expands the data (ratio > 1). Just above 45, every repeat becomes one token — the sweet spot. Far above it, the extra distance bits only bloat the output.
ratio = (tokens × (dist_bits + len_bits + 8)) / (chars × 8), the naive fixed-width triple model. Below 1.0 = compression; above 1.0 = expansion.
