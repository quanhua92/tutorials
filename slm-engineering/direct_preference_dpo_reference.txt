direct_preference_dpo_reference.txt - Web provenance log for the
DIRECT_PREFERENCE_DPO bundle. One [N] entry per URL, each with a "Verifies:"
line stating the exact claim/formula/value this source supports.

Every formula, year, author, and behavioral claim below is web-verified in
>=2 independent sources. Distinct URLs: 6.

================================================================================

[1] https://arxiv.org/abs/2305.18290
    Rafailov, Sharma, Mitchell, Ermon, Manning, Finn (2023) "Direct Preference
    Optimization: Your Language Model is Secretly a Reward Model" (NeurIPS 2023,
    the canonical DPO paper, official arXiv landing page).
    Verifies: the DPO loss itself, Equation 7:
      L_DPO = -E[ log sigma( beta*log(pi_th(y_w|x)/pi_ref(y_w|x))
                            - beta*log(pi_th(y_l|x)/pi_ref(y_l|x)) ) ]
      which (using log(a/b) = log a - log b) is exactly
      -log sigma( beta*(logpi_c - logpi_ref_c) - beta*(logpi_l - logpi_ref_l) )
      as implemented in direct_preference_dpo.py Section A. Also verifies:
      the implicit reward r_hat(x,y) = beta*log(pi_th(y|x)/pi_ref(y|x)) (eq.5,
      Section 5.1); the gradient structure that increases log-prob of y_w and
      decreases that of y_l (Section 4, "What does the DPO update do?"); and the
      claim that pi_ref = pi_SFT (the reference is the SFT checkpoint).

[2] https://huggingface.co/docs/trl/dpo_trainer
    HuggingFace TRL "DPO Trainer" documentation (the reference implementation
    used by Zephyr and most open DPO training). Authoritative secondary source.
    Verifies: the loss formula stated IDENTICALLY to eq.7:
      L_DPO(theta) = -E[ log sigma( beta*( log(pi_th(y+|x)/pi_ref(y+|x))
                                          - log(pi_th(y-|x)/pi_ref(y-|x)) ) ) ]
      with y+ = preferred (chosen), y- = dispreferred (rejected). Also verifies:
      "beta > 0 is a hyperparameter that controls the strength of the preference
      signal"; the logged metrics rewards/chosen = beta*log(pi_th(y+)/pi_ref(y+))
      and rewards/rejected = beta*log(pi_th(y-)/pi_ref(y-)); and the loss-type
      table (the default "sigmoid" loss is exactly eq.7).

[3] https://huggingface.co/docs/trl/v0.7.2/dpo_trainer
    HuggingFace TRL DPO Trainer docs (v0.7.2 archive) - explicit beta guidance.
    Verifies: "beta is the temperature parameter for the DPO loss, typically
    something in the range of 0.1 to 0.5. We ignore the reference model as
    beta -> 0." This is the source for the 0.1-0.5 typical-beta range used in
    Section C and the cheat sheet, and for the beta->0 behavior (loss -> log2,
    the policy stops anchoring to the reference).

[4] https://arxiv.org/html/2305.18290v3
    Rafailov et al 2023 (full HTML text of the DPO paper, v3). Primary source.
    Verifies (verbatim, with equation numbers):
      - Eq.3: the RLHF RL objective max_pi E[r(x,y)] - beta*KL(pi||pi_ref),
        the objective DPO replaces.
      - Eq.4: the closed-form optimal policy
        pi*(y|x) = (1/Z(x)) * pi_ref(y|x) * exp(r(x,y)/beta).
      - Eq.5: the reparameterized reward r(x,y) = beta*log(pi/pi_ref) + beta*log Z(x).
      - Eq.6: the preference probability in terms of policy + ref only
        (partition function cancels in the reward DIFFERENCE under Bradley-Terry).
      - Eq.7: the DPO loss (see [1]).
      - Section 4 gradient: dL/dtheta = -beta*E[ sigma(r_hat_l - r_hat_c) *
        (grad log pi(y_w) - grad log pi(y_l)) ] -> pushes chosen up, rejected down.
      - Section 4: "we initialize pi_ref = pi_SFT whenever available" (the
        reference IS the SFT model).

[5] https://arxiv.org/abs/2203.02155
    Ouyang, Wu, Jiang, Almeida, Wainwright et al (2022) "Training language
    models to follow instructions with human feedback" (InstructGPT, the
    canonical RLHF paper). THE lineage "old" DPO replaces.
    Verifies: the RLHF pipeline DPO replaces = (a) SFT, (b) train a SEPARATE
    reward model on preferences, (c) PPO against r(x,y) - beta*(log pi - log pi_ref)
    with a learned VALUE head. Confirms the "4 models in memory: policy + ref +
    reward + value" claim in Section E, and why the KL-to-reference term exists
    (keep the policy near pi_ref so the reward model stays accurate and the model
    does not mode-collapse).

[6] https://arxiv.org/abs/2310.16944
    Tunstall, Beeching, Lambert, Rajani et al (2023) "Zephyr: Direct Distillation
    of LM Alignment". The shipped-recipe DPO model in Section E.
    Verifies: Zephyr-7B-beta is trained with DPO (the "distilled DPO", dDPO)
    using TRL's DPOTrainer on the UltraFeedback dataset (64k prompts ranked by
    GPT-4), fine-tuned from Mistral-7B-v0.1 (the SFT base = the reference
    policy). The HF model card (HuggingFaceH4/zephyr-7b-beta) confirms the
    training metrics rewards/chosen, rewards/rejected, rewards/margins (the
    implicit-reward quantities r_hat = beta*log(pi_th/pi_ref)) and a final loss
    ~0.75, consistent with a single classification-style DPO objective with two
    models (policy + frozen ref). beta = 0.1 (the Zephyr alignment-handbook
    config; falls in the TRL-verified 0.1-0.5 typical range).
