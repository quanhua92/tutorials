Killer gotchas (verbatim from union_find.py Section D):
  1. Both optimizations: path compression (1 line in find) + union by rank (1 check in union). Either alone gives O(log n); both give O(α(n)).
  2. Rank grows only on a tie: rank[rx] += 1 ONLY when the two roots have EQUAL rank.
  3. 1-indexed nodes (P684): parent size n+1, initialized list(range(n+1)). Off-by-one is a silent bug.
  4. Count is maintained, not derived: starts at n, decrements ONLY on a successful union (union returned True).
  5. Two passes for equations (P990): union ALL == first, THEN check !=. Interleaving fails.
  6. Undirected only: union(x,y) is symmetric. For directed cycles / topo-sort use DFS coloring.
