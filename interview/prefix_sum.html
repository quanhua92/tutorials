A prefix sum is a running bank balance: save the total once, answer any range sum with one subtraction prefix[r] − prefix[l]. Add a hash map of past balances and every "contiguous subarray" problem drops from O(n²) to O(n). Step it yourself: watch the cumulative array build, the prefix-frequency map grow, and the left/right product passes fill the answer. Killer gotchas (verbatim from prefix_sum.py Section E):
  1. Missing empty prefix: for counting init freq={0:1}; for longest-by-index init first={0:-1}. Without it, every valid subarray starting at index 0 is dropped.
  2. Overwrite vs accumulate: counting uses freq[curr]+=1; longest uses if balance not in first — keep the earliest index only.
  3. Assign before multiply (Product Except Self): out[i]=running then running*=nums[i], else nums[i] leaks into its own answer.
  4. Half-open indexing: prefix has length n+1; sum[l:r)=prefix[r]-prefix[l].
  5. Negatives are fine for prefix sums & freq maps, but not for a sliding window.
  6. 0→−1 is the trick for "equal count of two values" — equal-count iff the running balance repeats.
