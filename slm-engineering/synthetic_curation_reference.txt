synthetic_curation_reference.txt -- web provenance log for SYNTHETIC_CURATION.md
================================================================================
Every formula, stage, count, and behavioral claim in SYNTHETIC_CURATION.md is
traced below to >=2 authoritative sources. This is the file the verification
sweep greps; "## Sources" in the .md is its public face.

The concept: pipelines for high-quality synthetic textbook & code dataset
generation (the Phi "Textbooks Are All You Need" -> Cosmopedia recipe). No
../llm/ lineage file exists for this bundle (per RESEARCH.md); all facts are
mined from the web below.


[1] https://arxiv.org/abs/2306.11644
    Gunasekar et al. 2023 "Textbooks Are All You Need" (phi-1, official arXiv)
    Verifies: phi-1 is a 1.3B-parameter Transformer trained 4 days on 8 A100s;
      data = 6B tokens "textbook quality" filtered web + 1B tokens synthetic
      textbooks/exercises generated by GPT-3.5 (= 7B tokens total). HumanEval
      pass@1 = 50.6%, MBPP = 55.5%. phi-1-small (350M) hits 45% on HumanEval.
      This is the founding evidence that "a little clean data >> a lot of dirty
      data" for SLMs. (Abstract, verified on the arXiv abstract page.)

[2] https://arxiv.org/abs/2309.05463
    Li et al. 2023 "Textbooks Are All You Need II: phi-1.5 technical report"
      (official arXiv, cited by the Cosmopedia blog as the target to replicate)
    Verifies: the authors "carefully selected 20K topics to seed the generation"
      of ~20 billion tokens of synthetic textbooks, and "use samples from web
      datasets for diversity" in the generation prompts. This is the SEED-TOPIC
      -> synthetic-textbook stage the Cosmopedia pipeline replicates.

[3] https://huggingface.co/blog/cosmopedia
    Ben Allal et al. 2024 "Cosmopedia: how to create large-scale synthetic data
      for pre-training" (Hugging Face official blog)
    Verifies: the core pipeline, multiple facts:
      (a) generator = Mixtral-8x7B-Instruct-v0.1; output = 30M+ files / 25B
          tokens -- "the largest open synthetic dataset to date".
      (b) the prompt has THREE components: the TOPIC, the SEED SAMPLE (a real
          web/curated page), and the GENERATION STYLE (audience + format).
      (c) prompt MULTIPLICATION: 4 audiences (young children, high school,
          college, researchers) x 3 styles (textbook, blog post, wikiHow) =
          "up to 12 times the number of prompts".
      (d) web-data path: clustered millions of web samples into 145 clusters,
          retained 112 as topics; web data = ">80% of the prompts" / 23M prompts.
      (e) benchmark DECONTAMINATION: "10-gram overlap" then difflib
          SequenceMatcher, discard if ratio > 0.5; applied across MMLU,
          HellaSwag, PIQA, SIQA, Winogrande, OpenBookQA, ARC-Easy, ARC-Challenge.
      (f) diversity target: "<1% duplicate content".
      (g) compute: "over 10k GPU hours".

[4] https://huggingface.co/blog/smollm
    Ben Allal et al. 2024 "SmolLM - blazingly fast and remarkably powerful"
      (Hugging Face official blog)
    Verifies: Cosmopedia v2 + SmolLM-Corpus mix:
      (a) Cosmopedia v2 = 28B tokens / 39M synthetic documents, still generated
          by Mixtral-8x7B-Instruct-v0.1.
      (b) v2 replaced v1's FineWeb clustering with a predefined list of 34,000
          topics using the BISAC book-classification standard; retrieved 1,000
          web pages per topic from 520M FineWeb samples.
      (c) v2 audience/style mix: 40% middle school, 30% college, 30% mix of
          other audiences/styles; +1B code-textbook tokens from AutoMathText.
      (d) SmolLM-Corpus = Cosmopedia v2 (28B) + Python-Edu (4B) + FineWeb-Edu
          dedup (220B); 135M/360M trained on 600B tokens, 1.7B on 1T tokens.
      (e) tokenizer vocab_size = 49152, embedding tying, context 2048 (ties to
          the 🔗 VOCAB_RATIONALIZATION and 🔗 SHARED_EMBEDDINGS bundles).

[5] https://arxiv.org/html/2502.02737v1
    Allal et al. 2025 "SmolLM2: When Smol Goes Big -- Data-Centric Training of
      a Small Language Model" (official arXiv HTML)
    Verifies: SmolLM2 is a 1.7B-parameter model; in its final pretraining mix
      the remaining mixture was "English web data at 58% ... and Cosmopedia v2
      ... at 4%". Confirms synthetic (Cosmopedia) is one (sizeable) source among
      several in a modern SLM data mix -- the 🔗 DATASET_MIXING bundle's premise.

[6] https://github.com/huggingface/cosmopedia
    Hugging Face "cosmopedia" GitHub repository (the open end-to-end pipeline)
    Verifies: the pipeline is shipped as runnable code (prompts + llm-swarm
      generation + text-clustering + datatrove dedup + lighteval). Confirms the
      stage order simulated in synthetic_curation.py: cluster -> seed-topic ->
      persona+style prompt -> generate -> filter+dedup.
