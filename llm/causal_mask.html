Causal Mask + QK-Norm — interactive CAUSAL_MASK.md causal_mask.py

Drag L (query len) and S (key len) and watch the causal triangle slide by k = S − L. Toggle prefill vs decode. Every mask cell is recomputed in JS from the identical formula in causal_mask.py.  |  the offset theme in RoPE →
What am I looking at?  A language model reads a sentence one token at a time. Each token may only read the notes written before it, never after. The grid below is the "bouncer" that enforces that rule: green = allowed (score 0), red = blocked (score −∞ → 0% probability after softmax). Row = the token asking the question; column = the note it wants to read.
mask is [L, S]; k = S − L = 0

① The causal mask grid [L, S] — green = attend (0), red = block (−∞)

Try this: set L=1, S=4 (decode) and watch the single row turn all-green — the lone new token may read its whole history. Now imagine the diagonal did not slide (k=0): that row would be one green then all red — the token cut off after the first word. That bug is silent garbage.
0 = ALLOWED (query attends) −∞ = MASKED (future position) the k-shifted diagonal
The triangle is built by tril(ones((L,S)), diagonal=S−L). In prefill (L=S) the diagonal sits at 0 → standard triangle. In decode (L=1, S=N) it slides right by N−1 → the single row becomes all-allowed.

② Prefill vs Decode — same formula, two regimes

Toggle prefill vs decode with the buttons above and watch the allowed/blocked grid shift its diagonal. Prefill = the first pass over the whole prompt (many tokens, L=S). Decode = generating one new token per step (L=1, S=the whole history so far). Same one-line formula, the diagonal just moves.
PREFILL   L = S, k = 0 → standard lower-triangle
DECODE   L = 1, S = N, k = N−1 → one row, all-allowed
Same causal_mask(L,S) call produces both. Forgetting the offset on decode (i.e. using k=0) collapses the new token's attention to key 0 only — silent garbage. (See CAUSAL_MASK.md §4.)

③ Masked attention on a fixed [B=1,H=1,L=4,S=4,D=8] input

What this shows: run the full masked attention once on a tiny fixed input. Because V is one-hot, the output numbers ARE the attention probabilities. Notice every cell strictly above the diagonal is exactly 0.0000 — the bouncer killed the future. Row l=0 is [1,0,0,0] (token 0 sees only itself); row l=3 spreads across all four keys.
PROBABILITIES probs[l,s] after softmax (masked cells are 0)
OUTPUT out[l,:] = probs · V (dims 4..7 are always 0)
Row l=0 is [1,0,0,0] (token 0 attends only to itself). Row l=3 spreads across all four keys. Every strictly-upper-triangular cell is exactly 0.

④ QK-Norm (Qwen3) — bound Q,K norms, keep softmax alive

What this shows: before comparing queries and keys, we quietly re-scale each one (RMSNorm) so their dot products don't blow up. Drag the input scale slider up and toggle QK-Norm off: watch max |score| shoot past 10 (softmax saturates → one-hot → gradients vanish → training stalls). Turn QK-Norm back on and the scores snap back to a bounded range.
RMSNorm on the D axis pins each Q/K vector's RMS to ~1 (so ‖q‖ = √D = 2.828), which caps the pre-softmax score magnitude. With big inputs and no QK-Norm the logits blow up to ±60 → softmax saturates → gradients vanish → training stalls. (See CAUSAL_MASK.md §5.)
