Git is the single source of truth. A controller (ArgoCD/Flux) running inside the cluster pulls desired state from Git, detects drift, and reconciles the cluster to match. Step through the reconcile loop and toggle auto-sync to see a manual edit get caught (or reverted). All states recompute live from the same model as gitops.py.

① GitOps flow — push to Git → controller pulls → cluster syncs

What to do: click step to walk through the reconcile loop: in-sync → dev pushes (drift) → controller syncs → in-sync again.

② Drift detection — manual kubectl edit → caught + corrected

What to do: toggle auto-sync. Someone runs kubectl scale web --replicas=5 (a manual edit NOT in Git). With auto-sync ON, the controller reverts it. With auto-sync OFF, it alerts and waits.

③ Pull vs Push — where the credentials live

What this shows: in push (traditional CI/CD) the CI server holds cluster creds and shoves manifests IN (needs inbound access). In pull (GitOps) the controller is INSIDE the cluster and reaches OUT to Git — no inbound access, creds stay sealed.
