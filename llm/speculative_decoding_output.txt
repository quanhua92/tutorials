========================================================================
  SECTION B: draft+verify pipeline (1 parallel target forward)
========================================================================
K = 4 draft tokens. Vocab V = 8.

STEP 1 - DRAFT (K serial forwards, but CHEAP - small model weights):
  draft(t_0)  -> t_0 ~ q(.)            1 cheap forward
  draft(t_1)  -> t_1 ~ q(.|t_0)        1 cheap forward
  draft(t_2)  -> t_2 ~ q(.|t_0,t_1)    1 cheap forward
  draft(t_3)  -> t_3 ~ q(.|t_0,t_1,t_2) 1 cheap forward

STEP 2 - VERIFY (1 EXPENSIVE forward - big model, but K tokens in parallel):
  target([t_0, t_1, t_2, t_3]) -> [p_0, p_1, p_2, p_3]

The causal mask (CAUSAL_MASK.md) makes position i attend only to 0..i:

         t_0    t_1    t_2    t_3
  t_0:  att     .      .      . 
  t_1:  att    att     .      . 
  t_2:  att    att    att     . 
  t_3:  att    att    att    att

  att = can attend,  .  = masked by causal mask
  Position i gets p_i(x | t_0..t_{i-1}) - the target's TRUE conditional.
  This is the SAME lower-triangular causal mask from CAUSAL_MASK.md,
  applied to the K-length verification sequence.

CONTRAST (why spec wins):
  sequential decode : 4 serial target fwds  = 4x weight loads (4x memory-bound)
  speculative verify: 1 parallel target fwd = 1x weight load  (Kx amortized)

[check] parallel verify uses the SAME causal mask as CAUSAL_MASK.md: OK

========================================================================
  SECTION C: rejection sampling verifier (accept + adjusted resample)
========================================================================
For each candidate t_i (draft prob q(t_i), target prob p(t_i)):
  1. Draw u ~ U(0,1)
  2. ACCEPT iff u <= min(1, p(t_i) / q(t_i))
  3. On REJECT at index R: discard t_R..t_K;
     resample t'_R from P_adj(x) = max(0, p(x) - q(x)) / Z

Per-token acceptance ratio min(1, p(x)/q(x)) for every vocab token:

========================================================================
  SECTION D: PROOF - output EXACTLY matches target distribution
========================================================================
THEOREM (Leviathan et al. 2023, Chen et al. 2023):
  Speculative decoding's output distribution is MATHEMATICALLY IDENTICAL
  to sampling directly from the target model p. Zero quality loss.

PROOF SKETCH (why P_adj makes it exact):
  P(output=x) = P(accept x) + P(reject) * P_adj(x)
             = q(x)*min(1,p(x)/q(x)) + Z * max(0,p(x)-q(x))/Z
             = min(q(x), p(x)) + max(0, p(x)-q(x))
             = p(x)   <-- EXACT

========================================================================
  SECTION E: speedup bound  S = (1-a^(K+1)) / ((1-a)(1+K*gamma))
========================================================================
  S = (1 - alpha^(K+1)) / ((1 - alpha) * (1 + K * gamma))

  alpha = acceptance rate (fraction of draft tokens the target accepts)
  K     = number of draft tokens per round
  gamma = draft_latency / target_latency  (how cheap the draft model is)

  Numerator   (1-a^(K+1))/(1-a) = 1+a+...+a^K = expected tokens committed/round
  Denominator 1+K*gamma           = relative cost: 1 target fwd + K draft fwds

========================================================================
  SECTION F: KV cache rollback on reject (KV_CACHE.md rewind)
========================================================================
During verification, the target model appends K draft tokens' K,V to the
cache. If rejection occurs at index R, tokens t_R..t_K are INVALID - their
K,V must be torn out. The KV cache rewind(n) does exactly this.

Dense cache demo (list-based, conceptually identical to TinyKvFullCache.rewind):

  Before spec round : cache_len = 3  (prefill tokens)
  After target verify: cache_len = 7  (prefill + K=4 draft tokens)
  Rejection at R=2  : rewind(2)  -> cache_len = 5
    (t_R=t_2..t_3 are INVALID; their K,V entries are discarded)

  -> The resampled token t'_R will be appended on the NEXT decode step,
     at its correct position via RoPE offset (ROPE.md §10).

Paged cache (vLLM/PagedAttention): rewind(n) returns physical pages to the
free list. The ceil-division off-by-one trap at exact page boundaries is
covered in KV_CACHE.md §F.

  [check] rejected count = K - R = 4 - 2 = 2: OK
  [check] rewind(K-R) conceptually matches DenseKVCache.rewind(2): OK

========================================================================
  SECTION G: variants - Medusa, EAGLE, Prompt-Lookup
========================================================================
All four use the SAME rejection-sampling verifier (Section C).
They differ only in HOW the draft tokens are generated:

| Method        | Drafting Mechanism                    | Verify Pattern       | Published |
|---------------|---------------------------------------|----------------------|-----------|
| Standard Spec | Small auxiliary autoregressive model   | Sequential chain     | 2-3x      |
| Medusa        | K independent heads on target's hidden | Tree attention       | ~2x       |
| EAGLE         | 1 autoregressive head on features      | Tree-structured      | 2.7-3.5x  |
| Prompt-Lookup | Heuristic n-gram match in input prompt | Standard chain       | varies    |

Standard : separate small model (e.g. Qwen-0.5B drafting for Qwen-72B).
           Pro: no target modification. Con: draft model sync/KV-cache overhead.

Medusa   : adds K extra linear heads to the target's last hidden state.
           Head i predicts token at position +i. No separate model, but heads
           are independent (weaker correlation -> lower alpha).
           Uses tree attention: verifies multiple candidate branches in parallel
           with a tree-structured causal mask.

EAGLE    : ONE lightweight autoregressive head operating on the FEATURE level
           (second-to-top-layer hidden state), fed the current token shifted by
           one step. Captures semantic context better -> higher alpha.
           Tree-structured verification with high-probability draft paths.

Prompt-  : zero-parameter heuristic. Matches the current context against the
Lookup     input prompt to find repeating n-grams. Excellent for summarization,
           QA, editing (where the output often copies/repeats the input).

Published figures (labelled, not measured here):
  Leviathan et al. 2023 (arXiv:2211.17192): 2x-3x on T5-XXL, identical outputs.
  Chen et al. 2023 (arXiv:2302.01318): 2x-2.5x on Chinchilla 70B.
  Medusa (arXiv:2401.10774): tree attention, multiple candidate branches.
  EAGLE (arXiv:2401.15077): 2.7x-3.5x on LLaMA2-Chat 70B, 2x throughput.
