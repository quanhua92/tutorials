Quicksort picks a pivot, splits the array into smaller | pivot | bigger (the partition), and recurses — in place. The whole game is the pivot choice: a good split is O(n log n); a bad one (smallest/largest on sorted input) degrades to O(n²). Step through a partition, then watch each pivot strategy on hostile input. The tiny worked example is recomputed in JS from the identical array as quick_sort.py.  |  heap_sort: the guaranteed O(n log n) sibling →

① The partition step — one linear scan, pivot lands in its final spot

What to do: press play (or step) and watch one partition of the worked array. The gold bar is the pivot (stashed at the end). The scanner j walks left→right; each value ≤ pivot is swapped into the growing small region (indices ≤ i). At the end the pivot is dropped between the two regions — locked into its final sorted position.
unscanned ≤ pivot (small region) pivot scanning j (≤ pivot) scanning j (> pivot) locked final pivot position
This is the Lomuto partition (Section A). The pivot's index after this pass is its FINAL place in the sorted array — the key fact that makes quicksort divide-and-conquer.

② Pivot strategies on the worked array — same algorithm, different pivot

What this shows: the four pivot rules, applied to the same array. Watch the top-level pivot they pick and the total comparisons each needs to fully sort it. median-of-3 spends 2–3 extra compares per call to dodge the worst case.
On this tiny random array the differences are small. On sorted/reversed input the gap explodes — see ③.

③ Pivot choice on hostile input — where O(n log n) becomes O(n²)

What to do: drag n. On sorted or reversed input, first/last pivot peels off one element per partition → n(n−1)/2 comparisons = O(n²) (red). median-of-3 and random stay near n log₂n on every input.
grey line = n log₂n (comparison-sort lower bound)
In words: the worst case is not exotic — it is the most common real-world input (already-sorted data). That is why every production quicksort uses median-of-3 and an introsort heapsort-fallback.
