📖 Read the full guide — architecture diagrams, decision tables, killer gotchas, cheat sheet. This page is the interactive companion.
Snapshot Isolation is fast but leaks write skew. SSI adds non-blocking SIREAD tripwires, watches for the dangerous structure (Ti -rw-> Tj -rw-> Tk, a pivot between two read-write edges), and aborts ONE transaction the moment the kink would complete. All graphs/detections below are recomputed in JS from the identical logic as serializable_ssi.py.  |  indexes tighten SIREAD range locks →

② Write skew — the doctors on call. Toggle SI vs SSI

The rule: at least one doctor must stay on call. Both read both rows (see 2 on call), each writes a different row to go off call. No ww-conflict, so SI lets both commit. SSI detects the 2-txn rw-cycle and aborts one.

③ Isolation-level hierarchy — click a column

Each level prevents a superset of the anomalies below it. PostgreSQL's REPEATABLE READ = SI, SERIALIZABLE = SSI. The lone gap is write skew.

④ Performance — SSI's abort-rate tax vs contention

Drag the keyspace K. Smaller K = hotter = more rw-overlap = more dangerous structures = more aborts. p_rw = r·w/K, abort ≈ P(in)·P(out), T_SSI = T0·(1−abort). N=20, r=2, w=1, T0=1000.
