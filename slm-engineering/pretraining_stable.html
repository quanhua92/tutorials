The thermostat problem: for a trillion-token walk you need (1) an LR schedule that peaks then decays, (2) decoupled weight decay (AdamW), and (3) a gradient clip that kills spikes. Drag the sliders below to reshape the cosine vs WSD curve, and to fire the gradient clip. Every number is recomputed in JS from the identical formula in pretraining_stable.py.  |  compare with micro pretrain eval →

① The LR schedule: cosine vs WSD — drag lr_max / warmup / decay_frac

How to read: both schedules ramp 0 → lr_max over warmup. Cosine then decays smoothly the WHOLE post-warmup span. WSD HOLDS lr_max flat, then decays only the final decay_frac (the "cooldown"). The GOLD pin is the cosine value at t=T/2=500 when warmup=0 — 1.65e-4.

② Gradient clipping: drag grad-norm & max_norm → watch the scale fire

How to read: scale = min(1, max_norm / ‖g‖). If ‖g‖ > max_norm the gradient is scaled down (direction kept, size capped to max_norm); else it's a no-op (scale = 1). The GOLD pin: ‖g‖=5.0, max_norm=1.0 → scale=0.2, clipped norm = 1.0.

③ What shipped models actually used (the cosine vs WSD eras)

How to read: GPT-3/Llama use AdamW + cosine + clip 1.0 + wd 0.1 — the de-facto standard since 2020. MiniCPM swaps cosine for WSD: same AdamW + clip, but holds lr_max ~90% of the run and decays the final 10%.
