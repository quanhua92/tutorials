NOTABLE CORRECTION TO THE ORIGINAL BRIEF (verified against primary sources):
  * The build brief cited FineWeb's dedup as "128 perms, 14 bands x 9 rows".
    This is INCORRECT on two counts:
      (1) The FineWeb paper (arXiv:2406.17557, NeurIPS 2024, appendix E.1)
          states VERBATIM: "we use 5-grams and 112 hash functions for our
          MinHash deduplication." 112 = 14 bands x 8 rows [refs 4, 5, 6].
      (2) 14 x 9 = 126, which is not 128 anyway -- the brief's own numbers were
          internally inconsistent. The correct production config is 14 bands of
          8 rows (112 permutations), NOT 128 / 14x9.
    The bundle uses the corrected 14 x 8 = 112 config in Section E.

[1] https://www.cs.princeton.edu/courses/archive/spring13/cos598C/broder97resemblance.pdf
    Broder, A. Z. (1997) "On the Resemblance and Containment of Documents",
    Proceedings. Compression and Complexity of SEQUENCES 1997 (COMPCOMM'97),
    IEEE, pp. 21-29, doi:10.1109/SEQUEN.1997.666900 -- the MinHash ORIGINAL.
    Verifies: the MinHash scheme and the core identity P(h_min(A) = h_min(B)) =
    J(A,B) = |A∩B|/|A∪B| (the unbiased Jaccard estimator this bundle's Section B
    implements); near-duplicate detection via shingling + resemblance; originally
    built for the AltaVista search engine to drop duplicate web pages.

[2] https://en.wikipedia.org/wiki/MinHash
    Wikipedia, "MinHash" (independent secondary source, cites Broder 1997 + Indyk).
    Verifies: Jaccard J(A,B) = |A∩B|/|A∪B|; the exact identity
    Pr[h_min(A) = h_min(B)] = J(A,B); the k-hash variant where the estimate is
    y/k (matches/length) with expected error O(1/sqrt(k)) (Chernoff bound);
    MinHash is an instance of locality-sensitive hashing; first applied in
    AltaVista for clustering/near-duplicate elimination of web documents.

[3] http://infolab.stanford.edu/~ullman/mmds/ch3n.pdf
    Leskovec, Rajaraman, Ullman, "Mining of Massive Datasets" ch.3
    "Finding Similar Items" (Stanford, 3rd ed.) -- the canonical textbook.
    Verifies: shingling (n-grams / w-shingling) as the document->set transform;
    MinHash signatures and P(sig_i equal) = J; LSH BANDING -- split the k
    signature into b bands of r rows (k = b*r), a pair is a candidate iff it
    agrees on all r rows of >= 1 band; the S-CURVE P(candidate | J) =
    1 - (1 - J^r)^b; the approximate THRESHOLD t ~= (1/b)^(1/r); the S-curve is
    steepest near the threshold (~0.5-0.63) and steplike.

[4] https://arxiv.org/abs/2406.17557
    Penedo et al 2024 "The FineWeb Datasets: Decanting the Web for the Finest
    Text Data at Scale" (Hugging Face, NeurIPS 2024 Datasets & Benchmarks,
    official).
    Verifies: FineWeb is a 15-trillion-token dataset from 96 Common Crawl
    snapshots; in-depth deduplication investigation (Section 3.4); appendix E.1
    "Deduplication parameters" states "we use 5-grams and 112 hash functions for
    our MinHash deduplication" (= 14 bands x 8 rows); dedup applied PER-SNAPSHOT.

[5] https://papers.neurips.cc/paper_files/paper/2024/file/370df50ccfdf8bde18f8f9c2d9151bda-Paper-Datasets_and_Benchmarks_Track.pdf
    FineWeb, NeurIPS 2024 official camera-ready PDF (the primary, archival text).
    Verifies: appendix E.1 verbatim "E Deduplication ... E.1 Deduplication
    parameters. As mentioned in Section 3.4, we use 5-grams and 112 hash
    functions for our MinHash deduplication." Independent confirmation (distinct
    URL) of ref [4]'s 5-gram / 112-permutation / 14-band / 8-row config.

[6] https://huggingface.co/spaces/HuggingFaceFW/blogpost-fineweb-v1
    Hugging Face, "FineWeb: decanting the web for the finest text data at scale"
    (official blog, companion to the paper).
    Verifies: the 15T-token corpus scale; MinHash near-duplicate dedup is a core
    pipeline stage; the dedup ablation showed removing it measurably HURTS
    downstream model quality (duplicates waste capacity + cause memorization) --
    the SLM-engineering motivation for this whole bundle.

[7] https://milvus.io/docs/minhash-lsh.md
    Milvus vector DB, "MINHASH_LSH" documentation (independent practitioner
    reference).
    Verifies: the banding math P(match in one row) = s, P(match in all r rows of
    a band) = s^r, P(no candidate) = (1 - s^r)^b, P(candidate) = 1 - (1 - s^r)^b
    -- the IDENTICAL S-curve this bundle's Section C prints; the threshold
    tuning tradeoff (precision vs recall) by choosing b and r.

[8] https://blog.gopenai.com/why-deduplication-is-the-most-underestimated-step-in-llm-pretraining-and-what-it-costs-you-to-get-a8d218f907a8
    GoPenAI, "Why Deduplication Is the Most Underestimated Step in LLM
    Pretraining" (2024-2025, independent practitioner analysis).
    Verifies: FineWeb's MinHash config restated as "5-gram shingling with 14 bands
    of 8 rows each (the equivalent of 112 hash permutations)"; the motivation that
    duplicates inflate loss estimates, waste training compute, and drive verbatim
    memorization -- reinforcing why dedup is the highest-leverage curation step.

SUMMARY OF VERIFIED NUMERICAL / ALGORITHMIC FACTS USED IN THE BUNDLE:
  Jaccard definition:   J(A,B) = |A∩B| / |A∪B|.                      [refs 1,2,3]
  MinHash identity:     P(h_min(A)=h_min(B)) = J(A,B).                [refs 1,2,3]
  MinHash estimator:    est J = (#matching sig positions)/k,          [ref 2]
                        expected error O(1/sqrt(k)).
  LSH banding:          k = b*r; candidate iff all r rows of a band    [refs 3,7]
                        agree in >= 1 of b bands.
  S-curve:              P(candidate|J) = 1 - (1 - J^r)^b.             [refs 3,7]
  Threshold:            t ~= (1/b)^(1/r).                              [ref 3]
  Toy gold anchor:      b=5, r=4 -> t = (1/5)^(1/4) = 0.66874.         [ref 3]
  FineWeb production:   5-grams, 112 perms, 14 bands x 8 rows.        [refs 4,5,6,8]
                        (NOT 128 / 14x9 -- corrected, see note above.)
  FineWeb threshold:    (1/14)^(1/8) = 0.7190 (~0.72).                 [derived, ref 3]
  FineWeb corpus scale: 15T tokens, 96 Common Crawl snapshots.         [refs 4,6]
  FNV-1a 32-bit:        offset basis 2166136261, prime 16777619        [standard]
                        (deterministic, used in place of Python's
                        randomized hash() so _output.txt reproduces).

UNVERIFIED / APPROXIMATE: none outstanding. The one correction to the brief
(FineWeb = 112 perms / 14x8, NOT 128 / 14x9) is settled by the primary NeurIPS
2024 paper appendix E.1 [refs 4,5] and corroborated by [6,8]. The FineWeb
threshold 0.72 is DERIVED from the verified 14x8 config via the standard
threshold formula t ~= (1/b)^(1/r) [ref 3]; FineWeb does not print this number
directly, but it follows from the config they do print.
