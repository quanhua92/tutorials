Killer gotchas (verbatim from backtracking.py Section E):
  1. Aliasing: results.append(path) stores a reference — later .pop() mutates every entry. Always append a copy: path[:].
  2. Missing un-choose: forget path.pop() and choices leak into siblings. Every choose needs a matching unchoose.
  3. Wrong start index: combos/subsets recurse with i+1; Combination Sum recurses with i (reuse). Swapping yields dupes or misses.
  4. Pruning unsorted input: break when cands[i] > remaining only works after sorting. SORT FIRST.
  5. Duplicates (P090): sort + if i > start and nums[i] == nums[i-1]: continue — the i>start guard dedupes siblings only.
  6. Subsets vs Combinations: subsets records at every node; combinations records only when len(path) == k.
