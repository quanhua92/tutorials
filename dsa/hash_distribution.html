A hash table is M buckets; the hash function decides which bucket each key lands in. A good one fills every bucket evenly, scrambles bits so input structure vanishes (the avalanche effect), and sees collisions at exactly the rate the birthday paradox predicts. Every number below is recomputed live in your browser with the identical formulas and seeded key generator as hash_distribution.py — the green badges audit it.  |  read the guide →

① Bucket distribution — 1000 keys into 16 buckets (uniform line = 62.5)

What to do: pick a hash, then toggle the key source. Random keys fill every bucket near 62.5 for all four. But feed keys that are multiples of the prime (step = 1 000 003) and watch mod_prime collapse — every key divides evenly, so key % prime == 0 and all 1000 pile into one bucket. The real hashes don't care.
chi-squared = Σ (observed − 62.5)² / 62.5; mean for 16 buckets ≈ 15. Lower and closer to 15 = flatter. mod_prime on step=prime scores ~15000 — a catastrophic, predictable failure (HASH_DISTRIBUTION.md §5).
