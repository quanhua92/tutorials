Gradient Checkpointing (Activation Recomputation) — interactive GRADIENT_CHECKPOINTING.md gradient_checkpointing.py

The suitcase idea: backprop is a round trip — drop a snack (activation) at every layer on the way out, eat each on the way back. store-all carries all L snacks (O(L) memory); full-recompute carries none but walks back to the start each time (O(L²) compute); selective √L stashes a snack every √L-th layer — O(√L) memory for one extra forward (+33% compute). Drag L and watch the grid + bars redraw. Every number is recomputed live in JS from the identical formula in gradient_checkpointing.py (a faithful activation-memory simulation; the memory & compute arithmetic is exact).  |  DDP · ZeRO (shard the model+optimizer 20N — the other memory axis) · Pipeline Parallel (1F1B cuts in-flight micro-batch activations) · MLP block (the per-layer internals checkpointing discards)

① The checkpoint grid — which layers are STORED (■) vs RECOMPUTED (□)

What to do: drag L (depth). Green ■ cells are the checkpoints whose input activation is KEPT (their index set matches gradient_checkpointing.py Section D). Faint cells are thrown away and recomputed in backward. The default count is ceil(√L) (Chen 2016's optimum); drag the checkpoints slider to override it and watch only the memory move — compute stays at one extra forward.
STORED checkpoint (keep its input activation) recomputed in backward (discarded, redone)

② Activation memory across the 3 strategies (log scale)

How to read: each bar is the memory multiplier (× one layer's activation A). store-all = L (linear in depth); full-recompute = 1 (only the input); selective = #checkpoints ≈ √L. Log scale so the small bars stay visible. Drag L — store-all races away while selective crawls as √L.

③ Compute cost — selective is +33%, full-recompute is O(L²)

How to read: compute in layer-forward units (backward ≈ 2× forward). store-all = 3L (1 fwd + 1 bwd). selective = 4L — one extra forward → +33% (the dashed line). full-recompute = L + L(L+1)/2 + 2L = O(L²) — it explodes with depth. This is why nobody trains full-recompute: the √L trick is the Pareto pick.
