Killer gotchas (verbatim from greedy.py Section D):
  1. Sort by END, not start: sorting by start fails on nested intervals ([1,10] hides [2,3]). The earliest-finishing interval leaves the most room.
  2. Global check is mandatory (Gas Station): resetting start is local — you MUST still confirm total >= 0 or you return a bogus start for an impossible circuit.
  3. Strict vs non-strict: P452 uses start > arrow_pos (touching shares an arrow); P435 uses start >= end. Same skeleton, different off-by-one.
  4. Greedy is not always correct: coin change [1,3,4] target 6 → greedy gives 4+1+1 (3) but 3+3 (2) is optimal. The exchange argument proves safety.
  5. Jump Game needs only the envelope: never store which jumps to take — just max_reach. A DP wastes O(n²) time on an O(n) problem.
  6. The -1 in Task Scheduler: (max_freq-1)·(n+1)+count_max — the last execution needs no trailing cooldown.
