2. LRU vs LFU vs FIFO — same workload, different eviction

One workload at capacity 2 (warm key 1 three times, touch key 2, then insert key 3 to force an eviction). The table mirrors lru_cache_output.txt (Section “LRU vs LFU vs FIFO”). LFU protects the hot key; LRU and FIFO evict it.

policyevicted @ put(3)final membershitshit rate

Reading: on a frequency-skewed workload (a hot key) LFU wins because it remembers access counts; on a recency-skewed workload (temporal locality, e.g. a news feed) LRU wins. FIFO ignores access entirely — cheapest, rarely right for a value cache.

3. Reference — the four helpers & five invariants

The whole algorithm is four pointer primitives. Every public method composes them.
