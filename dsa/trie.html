Trie (Prefix Tree) — interactive TRIE.md trie.py

Each edge is one character; the path root → node spells a key. Type a word to watch the O(L) search walk; type a prefix to harvest every completion. Toggle the radix tree to see single-child chains collapse into substring edges.  |  Segment tree →

① The trie for [cat, car, card, care, dog]

What to do: toggle standard / radix to see how the Patricia trie compresses the d→o→g chain into one edge "dog". * marks a node where a complete key ends.
internal node key-end node (*) edge label (character / substring)

② Search — walk one edge per character, then check the end flag (O(L))

What to do: type a lowercase word. The blue path shows the O(L) walk; a green * means the key is present (the final node is an end-node). Try card (present) and cab (misses at 'b').
L = key length; the walk stops at the first missing edge.

③ Prefix query — walk to the prefix node, then harvest its subtree (O(L + M))

What to do: type a prefix. The gold node is the prefix node; every green * in its subtree is a completion. This is what a hash table cannot do without scanning all keys.
M = number of matches returned.
