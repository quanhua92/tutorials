GIN & GiST — specialized indexes for non-trivial data GIN_GIST.md gin_gist.py

Two opposite strategies for data a B-tree can't handle. GIN builds an inverted index (element → TID list) — type words to intersect posting lists. GiST builds a tree of bounding-box union keys — click the map to descend and prune subtrees. Recomputed live in JS from the identical deterministic inputs as gin_gist.py.  |  hash index (equality) →

① GIN — the inverted index: element → posting list, intersect for AND

What to do: type space-separated words. GIN fetches each word's posting list (the TIDs of docs containing it) and intersects them. Highlighted rows are the queried elements; green chips are the docs surviving the intersection.

② GiST — bounding-box tree: click to query, watch subtrees get pruned

What to do: click anywhere on the map (or drag the sliders) to set the query point. GiST tests each node's union key (bounding box): contains the point → descend; misses → prune the whole subtree. Gold rectangle = a hit.

A B-tree cannot index "contains point" (no total order on 2D). GiST turns an O(N) sequential scan into O(log N) by skipping every subtree whose box misses the point — here it visits 3 of 7 nodes for either query.
