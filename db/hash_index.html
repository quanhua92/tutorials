The Hash Index (PostgreSQL linear hashing) — interactive 📖 HASH_INDEX.md 📄 hash_index.py

📖 Read the full guide — architecture diagrams, decision tables, killer gotchas, cheat sheet. This page is the interactive companion.
Each key is hashed (FNV-1a stand-in for PG's hash_any) to a 32-bit value and routed to a bucket. When a bucket's primary page fills, an overflow chain grows. When the load factor crosses 0.75, ONE bucket splits (Litwin 1980 linear hashing) — items rehash, some move to a new bucket. Slide # keys to watch splits and chains appear; pick a key to trace its lookup down the chain. Recomputed live in JS from the identical inputs as hash_index.py.  |  B-tree side of the comparison →
