The weight-stays, activation-flies idea: a 671B MoE model is ~335 GB even at 4-bit (🔗 QUANTIZATION) — way beyond a 24 GB GPU. Layer offload streams all 350 GB of weights across PCIe every token (~0.18 tok/s — unusable). KTransformers parks the MoE expert weights in CPU DRAM and runs only the k active experts via Intel AMX (🔗 MOE_ROUTING), so per token only the tiny ~14 KB activations cross PCIe — not the weights. Every number is recomputed in JS from the identical formulas in ktransformers_offload.py. ① The GPU + CPU split — where each part of a MoE layer lives

How to read: a MoE layer (🔗 MOE_ROUTING) is physically split. The small, hot parts (attention, router, shared expert, KV cache — 🔗 PAGED_ATTENTION) stay on the GPU; the huge, cold routed expert weights are parked once in CPU DRAM. Per token, only the tiny hidden state flies across PCIe — the weights never move.
GPU VRAM (small, hot, every token) CPU DRAM (huge, cold, parked once) activation flies across PCIe (~14 KB/call) AMX GEMM on the k active experts

② The gold centerpiece — 350 GB weights vs ~14 KB activations (log scale)

What to do: drag model size, hidden, PCIe gen. The red bar (layer offload: all weights/token) is always thousands to millions of times taller than the green bar (expert offload: only activations). At DeepSeek-V3 defaults (350 GB, hidden 7168, Gen4): layer offload = 5.47 s/tok (0.18 tok/s); expert offload full = 219 µs/tok — a ~25,000× transfer reduction. See KTRANSFORMERS_OFFLOAD.md §7.
