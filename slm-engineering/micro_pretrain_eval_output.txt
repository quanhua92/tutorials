==========================================================================
  SECTION B: Pearson r between each slice and the benchmark
==========================================================================
For each slice s, r = cov(L_s, B) / (sigma_Ls * sigma_B).
|r| near 1 => the slice is a FAITHFUL cheap proxy for the benchmark.

| slice        | Pearson r |  |r|  | faithful? (|r| >= 0.85) | verdict                |
|--------------|-----------|------|----------------------|------------------------|
| code         |   -0.9976 | 0.9976 | YES                  | faithful proxy         |

| factual      |   -0.8764 | 0.8764 | YES                  | faithful proxy         |
| reason       |   -0.9975 | 0.9975 | YES                  | faithful proxy         |

Best single slice: 'code' with r = -0.9976  (|r| = 0.9976).
All four r values are NEGATIVE because slice loss falls while the
benchmark rises. The MAGNITUDE |r| is what matters for prediction.

==========================================================================
Fit on the early checkpoints (indices 0..4, steps 2000..10000):
  a (slope)     = Sxy/Sxx = -0.252461

==========================================================================
  SECTION E: the decision recap -- which eval, when
==========================================================================
| strategy            | how                 | what it buys you        | use when                  |
|---------------------|---------------------|-------------------------|---------------------------|
| Full bench / ckpt   | all 8 benchmarks x every ckpt | ground truth, weeks of GPU | Pythia-style final analysis only |
| Held-out val loss   | one aggregate cross-entropy | free (one forward pass), weak tie to capability | cheap overfitting watch (GPT-3 style) |
| Slice correlation   | per-capability loss + Pearson r | shows WHICH capability is moving | diagnostic mid-train      |
| MICRO-BENCH         | {code} slice(s) + fitted line | seconds/ckpt, predicts benchmark to <2% | daily training loop       |

The single question that picks the row:
  IS THIS THE FINAL CHECKPOINT?         -> full bench (the truth).
  DO YOU JUST NEED AN OVERFITTING WATCH? -> held-out val loss.
  DO YOU NEED PER-CAPABILITY SIGNAL?     -> slice correlation.
  IS THIS A DAILY MID-TRAIN CHECK?       -> MICRO-BENCH (code).

  [check] the four strategies are distinct in how the eval is done: OK
