Killer gotchas (verbatim from bit_manipulation.py Section D):
  1. Power-of-two trap: always n > 0 and (n & (n-1)) == 0 — 0 wrongly passes without the guard.
  2. n & (n-1) is NOT n - 1 — it clears the lowest set bit, keeps every other bit.
  3. Python's ~ is -(n+1), not "flip bits". Build a mask: (1 << bit_length) - 1.
  4. XOR swap aliasing: a ^= a sets a to 0. Never swap when indices might be equal.
  5. Shift ≥ word size is UB in C/C++; Java wraps mod 32. Python just works (bignum).
  6. "Every element k times except one": count bits per position mod k — O(1) space, no HashMap.
