① Two documents → MinHash signatures → estimated Jaccard (drag the text)

How to read: each document becomes a set of 3-word shingles, then a k=20 MinHash signature (entry i = min over shingles of hi, with hi(s)=fnv1a(i+"|"+s) — deterministic, identical to the .py). The fraction of matching signature positions unbiasedly estimates Jaccard: P(sigA[i]==sigB[i]) = J. Edit doc_b and watch the estimate track the true overlap.
What to do: change one word in doc B (e.g. bank → shore, fox → hound, jumps → leaps) and watch the exact J, the estimate, and the match count move together. A single end-word edit changes ONE shingle (high J ≈ 0.83); a middle-word edit changes THREE (lower J ≈ 0.57).

② LSH banding — drag (b, r) → the S-curve threshold moves

How to read: with b bands of r rows (k = b·r), a pair is a candidate iff it agrees on all r rows of ≥1 band. The probability is the S-curve P(candidate|J) = 1 − (1 − J^r)^b — ~0 below the threshold t ≈ (1/b)^(1/r) and ~1 above it. Raise r → higher threshold (more precision, more false negatives); raise b → lower threshold (more recall, more false positives). FineWeb ships b=14, r=8.
k = b·r = 20  ·  threshold t = 0.6687
