The order is fixed: Huffman codes the symbols LZ77 emits, so LZ77 must run first. gzip = DEFLATE body + gzip header/trailer; zlib = DEFLATE body + zlib header/trailer. Same body, different bookends.

② Pass 1 step-through — literals become back-references

What to do: step through the encoder. The first TOBEORNOT (9 bytes) is emitted as literals — nothing to copy yet. After that, every recurrence is a back-reference: match(len, dist) = “go back dist, copy len bytes.” Blue cells are the copy source.

The overlap trick: a back-reference may copy from inside the bytes currently being produced. match(len=7, dist=1) on aaaaaaaa copies 1 byte ago, 7 times — the output grows as it copies. This is how LZ77 expresses long runs.

③ Pass 2 — Huffman codes + the final bit string (recomputed live)

What this proves: the page rebuilds the Huffman trees from the token frequencies and packs the whole stream in your browser, on the same input as deflate.py Section E. The green check: OK confirms every number matches the .py exactly.
Bit-packing caveat: real DEFLATE mixes bit orders (Huffman codes high-bit-first, extra bits low-bit-first). This page packs every field high-bit-first for readability, so the bytes are not gzip-identical — but the algorithms (LZ77 match-finding, Huffman construction, the RFC 1951 length/distance code tables) are faithful, and the round-trip is exact.
