② The call-count dial — Volcano makes N calls, vectorized makes N/batch

What this shows: for a given operator, Volcano calls it once per tuple (N); vectorized calls it once per batch (N/batch). Drag the batch size: at batch = N the filter drops to a single call.
vectorized_calls_per_op = ceil(N / batch). At batch = N every operator is a single call. Real engines pick batch ≈ 1024–2048: large enough to saturate SIMD, small enough to stay in L1 cache.

④ Late materialization — when is column b fetched?

What to do: toggle early vs late. Early reads all of column b up front (wasted on rejected rows); late reads b only at the surviving indices.

Lower selectivity = bigger win. On a 10-col table at 1% selectivity, late materialization reads ~89% less I/O than early. Impossible in a row store.
