Root index.json field → what it controls

Every root field binds to something outside the data file (render dims, caption CSS, audio tracks, sequencing). None carry animation — that is index.html's job.
FieldType / shapeControls
canvas{width, height, fps} numbersHF render + ffmpeg encode dims/fps (RFC §10). Default 1920×1080@30.
theme.caption_styleenumWhich caption CSS variant the caption pipeline emits. One of highlight | neon | editorial | eco-green.
theme.colors / fontsobject mapsBrand palette + font stacks injected into slides.
audio.music{asset, volume, loop}Music bed track. asset is a sha256: content ref; volume 0..1 (default 0.08); loops.
audio.voiceover{asset, auto_generated}Voiceover track. asset is a filename (TTS-baked); no loop — plays once, drives captions.
transition_default{type, duration}Default between-slide transition (root-owned). A slide may override it.
slidesstring[] (ordered ids)The sequencing spine the root index.html host reads. ids resolve to slide unit folders.

Schema (RFC 0001 §5.2, verbatim)

// root index.json — project-wide DATA (not per-slide content) { "version": 1, "canvas": { "width": 1920, "height": 1080, "fps": 30 }, "theme": { "caption_style": "highlight", "colors": {}, "fonts": {} }, "audio": { "music": { "asset": "sha256:...", "volume": 0.08, "loop": true }, "voiceover": { "asset": "voiceover.mp3", "auto_generated": true } }, "transition_default": { "type": "crossfade", "duration": 0.4 }, "slides": ["slide-0", "slide-1", "slide-2"] }
