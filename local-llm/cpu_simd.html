CPU SIMD — 8 weights, one instruction 📖 CPU_SIMD.md 📄 cpu_simd.py

One SIMD instruction does the work of N scalar ops. The viewer below shows 8 Q4_0 nibbles flowing into 8 AVX2 lanes simultaneously — dequant → mul → accumulate in one pass, while scalar grinds them one at a time. The block size of 32 (see quant_types) is not an accident: 32 × 4 bits = 128 bits matches the register width.

① SIMD lane viewer — 8 weights × 8 lanes, one instruction per phase

What this shows: the 8 Q4_0 nibbles enter 8 AVX2 float32 lanes at once. Press step to walk through dequant → mul → accumulate. Each phase is ONE vector instruction touching all 8 lanes in parallel. The "scalar" mode highlights one cell at a time — the same math, 8× the instructions.

In AVX2 mode each phase lights all 8 cells at once (1 instruction). In scalar mode cells light one-by-one (8 instructions per phase). The final dot product is identical — SIMD changes the op COUNT, not the math.

② Instruction count — same dot product, three ISAs

What this shows: the dot product of 8 Q4_0 weights × 8 inputs (= −44.0 on the gold block) computed three ways. AVX-512 collapses it to ~7 vector instructions; scalar needs 15. Each SIMD instruction replaces N scalar ops, where N = register_bits / 32.
Speedup is computed against the no-FMA scalar baseline (15 ops). With FMA, scalar drops to 8 ops but SIMD still wins — the per-instruction work is multiplied by the lane count.

③ ISA comparison — register width, lanes, availability

What this shows: the four SIMD ISAs you will actually hit running local LLMs. The float32-lanes column drives the dot product; the Q4-elem/lane column drives the unpack shuffle. Apple Silicon is NEON-only but compensates via unified-memory bandwidth.
Block size 32 = 128 packed bits = half an AVX2 ymm = one NEON Q register. The block layout was designed for the silicon.
