Maximum Flow (Ford-Fulkerson + Edmonds-Karp) — interactive MAX_FLOW.md max_flow.py

A flow network is plumbing: each pipe has a capacity, and we want the most water from source s to sink t. Ford-Fulkerson repeatedly finds an augmenting path in the residual graph and pushes the bottleneck. Edmonds-Karp picks the path by BFS (shortest). The network below is the canonical CLRS example, recomputed live in JS to match max_flow.py exactly.

① The flow network — step through the 3 augmenting paths

What to do: press step ▶. Each round BFS finds the shortest s→t path with residual > 0 (the green arrows), computes the bottleneck (the tightest pipe), and pushes it. Reverse residuals (dashed) open up so the method can "undo" later. After 3 augmentations no path remains → max flow = 23.

② The residual update — forward room shrinks, reverse room grows

Pushing bottleneck b along path edge (u,v) does: r(u,v) -= b (less room forward) and r(v,u) += b (more room to undo). The reverse residual is the genius of Ford-Fulkerson — it is how the method later cancels flow that turned out to be a bad idea, converging to the true optimum.
Switch the dropdown above to residual graph to see all residual edges drawn: forward residuals as solid lines, reverse residuals (undo room) as dashed purple.

③ Max-flow = min-cut — the theorem in one picture

At termination, BFS from s in the residual graph reaches exactly the S-side of the minimum cut. The cut edges (A→B) are all saturated, and their capacity sum equals the max flow. Switch the dropdown to min-cut (or it auto-selects at iteration 3) to see the partition.

④ Applications — what max-flow actually solves

The unifying trick: model "choices" as a cut. The min cut picks the cheapest consistent partition; the max flow proves it is optimal via the max-flow min-cut theorem.
