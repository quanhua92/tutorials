==========================================================================
  SECTION B: gradient clipping by global norm (scale = min(1, max_norm/||g||))
==========================================================================
Clipping caps the global gradient norm so one bad batch can't catapult
the weights. DIRECTION is preserved (every grad scaled by the SAME
factor); only the SIZE shrinks.

--- (1) grad with norm 5.0, clip to max_norm 1.0 (the GOLD anchor) ---
  g        = [3.0, 0.0, 0.0, 0.0, 4.0]
  ||g||    = sqrt(3^2 + 4^2) = 5.0000
  scale    = min(1, max_norm/||g||) = min(1, 1.0/5.0000) = 0.2000
  g' = g*scale = [0.6, 0.0, 0.0, 0.0, 0.8]
  ||g'||   = 1.0000   (clipped to exactly max_norm)

GOLD PIN (pretraining_stable.html recomputes this identically):
  scale (||g||=5, max_norm=1) = 1.0/5.0 = 0.2 ; clipped norm = 1.0
  [check] GOLD clip scale == 0.2 (||g||=5, max_norm=1): OK
  [check] GOLD clipped norm == 1.0: OK
  [check] original norm was 5.0 (3-4-5 triangle): OK

--- (2) grad with norm 0.5 (< max_norm 1.0) -> UNCHANGED (scale = 1) ---
  ||g|| = 0.5000 < max_norm=1.0 -> scale = min(1, 1/0.5000) = 1.0000
  ||g'|| = 0.5000  (no change; below the clip threshold)
  [check] small grad (< max_norm) is unchanged (scale == 1): OK

--- (3) multi-tensor GLOBAL norm (a 2-tensor case, as in real models) ---

==========================================================================
  SECTION C: AdamW (decoupled) vs Adam+L2 (coupled) -- 1-step contrast
==========================================================================
Same parameter w, same gradient g, same wd. The ONLY difference is WHERE
weight decay enters: AdamW shrinks w DIRECTLY; Adam+L2 folds wd*w into g
and then lets the momentum average it -> the decay is diluted.

w0=1.00, g=0.0100, wd=0.1, lr=0.01, beta1=0.9, beta2=0.95, t=1

AdamW:    w' = w - lr*(m_hat/(sqrt(v_hat)+eps) + wd*w)
          adaptive step = 0.010000 (lr * sign(g) at t=1)
          decay term    = lr*wd*w = 0.001000
          w' = 1.0000 - 1e-02*(step + 0.0010) = 0.989000

Adam+L2:  g' = g + wd*w = 0.0100 + 0.1*1.0 = 0.1100  (L2 folded INTO gradient)
          w' = w - lr*m_hat/(sqrt(v_hat)+eps)  (no separate decay)
          w' = 1.0000 - lr*0.1100/(|0.1100|) = 0.990000

weight shrinkage:  AdamW = 0.011000 ,  Adam+L2 = 0.010000
  AdamW shrinks w MORE here (0.011000 > 0.010000) because the
  decay term acts on the FULL weight regardless of the tiny gradient.
  Adam+L2's decay was averaged into the moment, so it behaves like a
  scaled-down gradient step -> the effective decay is smaller.

  [check] AdamW != Adam+L2 for the same wd (the decoupling changes the update): OK
  [check] AdamW applies a strictly larger decay-driven shrink here: OK
  [check] both steps move w in the correct direction (toward smaller |w|): OK

--- the high-momentum case (where Adam+L2's dilution shows) ---
Pre-load momentum so t is large: beta1^t -> 0, bias correction -> 1.
A param with a STEADY large gradient builds high momentum; under Adam+L2
the decay is averaged with that big gradient, so it barely dents the
update. AdamW shrinks the weight by the SAME lr*wd*w regardless.

w0=2.0, g=1.0, m=9.0 (warm), t=1000
  AdamW   w' = 1.959610   (decay = lr*wd*w = 0.0020 added)
  Adam+L2 w' = 1.961609   (decay averaged into m, diluted)
  [check] AdamW shrinkage > Adam+L2 shrinkage at high momentum (the decoupling win): OK

That dilution is the bug AdamW fixes: a coefficient 'wd' should mean the
same thing for every parameter, regardless of its momentum history.

==========================================================================
  SECTION E: lineage recap + what shipped models actually used
==========================================================================
The optimization recipe improved four times; each step removed a failure
mode:

| stage         | what it does                       | failure removed            | era                     |
|---------------|------------------------------------|----------------------------|-------------------------|
| SGD           | one LR for all params; no momentum | fragile on transformers    | convex / pre-Adam       |
| Adam          | adaptive per-param rates from m, v | no built-in regularization | the adaptive baseline   |
| Adam+L2       | decay folded into g (lambda*w)     | decay diluted by momentum  | the coupled bug         |
| AdamW         | DECUPLE: decay direct on w (wd*w)  | stable, default for all LLMs | Llama, GPT-3 era        |
| + cosine/WSD  | warmup -> peak -> decay to lr_min  | settles into a sharp min   | the modern schedule     |
| + grad clip   | cap ||g|| <= max_norm each step    | kills loss spikes -> no NaN | Llama 2 / GPT-3 clip=1.0 |

What shipped models actually used (all web-verified; see _reference.txt):

| model    | optimizer | weight decay | grad clip | LR schedule                | source                          |
|----------|-----------|--------------|-----------|----------------------------|--------------------------------|
| GPT-3    | Adam      |          0.1 |       1.0 | cosine+warmup              | Brown 2020 arXiv:2005.14165 (sec 4) |
| Llama 1  | AdamW     |          0.1 |       1.0 | cosine, 2000 warmup, peak 3e-4 | Touvron 2023 arXiv:2302.13971  |
| Llama 2  | AdamW     |          0.1 |       1.0 | cosine, 2000 warmup, peak 3e-4, ->10% peak | Touvron 2023 arXiv:2307.09288 (train A.1) |
| MiniCPM  | AdamW     |          0.1 |       1.0 | WSD (10% decay)            | Hu 2024 arXiv:2404.06395       |

Two patterns:
  * GPT-3/Llama use COUPLING-FREE AdamW + cosine + clip=1.0 + wd=0.1 -- the
    de-facto standard since 2020. The cosine decays to ~10% of the peak.
  * MiniCPM swaps cosine for WSD: same AdamW + clip, but holds lr_max for
    ~90% of the run and decays only the final 10%. The stable-phase
    checkpoint is reusable for continual pretraining.
