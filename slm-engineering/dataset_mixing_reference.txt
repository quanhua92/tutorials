[1] https://arxiv.org/abs/2305.10429
    Xie, Pham, Dong, Du, Liu, Lu, Liang, Le, Ma, Yu (2023).
    "DoReMi: Optimizing Data Mixtures Speeds Up Language Model Pretraining."
    NeurIPS 2023 (official, DeepMind/Stanford).
    Verifies:
      - Domain Reweighting with Minimax Optimization (Group DRO over domains)
        sets domain weights WITHOUT downstream-task knowledge.
      - A 280M-parameter proxy model sets the domain weights for training an
        8B-parameter model (30x larger).
      - On The Pile: "improves perplexity across all domains, even when it
        downweights a domain" -- i.e. it reduces the WORST-domain loss.
      - "+6.5% average few-shot downstream accuracy" over The Pile's default
        domain weights; "reaches the baseline accuracy with 2.6x fewer
        training steps."
      - On GLaM: matches performance of weights tuned ON downstream tasks.
      - The mechanism: a reference policy sets expected per-domain losses;
        the bandit raises weights on domains whose loss EXCEEDS the reference
        prediction (the "excess loss" signal used in Section C of dataset_mixing.py).

[2] https://arxiv.org/abs/2403.16952
    Kang, Sun, Jiang, Chen, Huang, Camburu (2024).
    "Data Mixing Laws: Optimizing Data Mixtures by Predicting Language Modeling
    Performance." ICLR 2025 (official).
    Verifies:
      - The mixture proportions of pretraining domains (web, code, papers, ...)
        "crucially impact" downstream performance, and this impact can be
        QUANTITATIVELY PREDICTED from small-scale experiments.
      - Predicts language-modeling loss (and downstream proxies) as a function
        of the data mixture ratios -- the theoretical justification for treating
        alpha as an optimizable variable rather than a guessed hyperparameter.
      - Confirms the bundle's central thesis: the mix is as important as scale.

[3] https://arxiv.org/abs/2203.15556
    Hoffmann et al. (2022). "Training Compute-Optimal Large Language Models."
    (Chinchilla, official, DeepMind.)
    Verifies:
      - Chinchilla was trained on the MassiveText dataset (same as Gopher);
        the data mixture is a FIXED heuristic mix of web/books/C4/news/GH/Wiki
        (the Section D "Chinchilla" row).
      - "Current large language models are significantly undertrained" -- the
        motivation for treating data quantity AND mixture quality as
        first-class scaling variables (not just parameter count).
      - Compute-optimal split scales N and D together; D_opt ~= 20*N.
        (Cited as the compute backdrop against which data-mixing acts.)

[4] https://arxiv.org/abs/2302.13971
    Touvron et al. (2023). "LLaMA: Open and Efficient Foundation Language
    Models." (official, Meta AI.)
    Verifies: the Llama pretraining data mixture (Section D "Llama-1" row).
      - English CommonCrawl: 67%
      - C4: 15%
      - GitHub: 4.5%
      - Wikipedia: 4.5%
      - Books: 4.5%
      - ArXiv: 2.5%
      - StackExchange: 2%
    (Sum ~= 100.5%; the canonical published heuristic mix.)

[5] https://arxiv.org/abs/2502.02737
    Lozhkov, Bakouch, Penedo, Tunstall et al. (2025).
    "SmolLM2: When Smol Goes Big -- Data-Centric Training of a Small Language
    Model." ICML 2025 (official, Hugging Face).
    Verifies: the SmolLM2 multi-stage mixture (Section D "SmolLM2 stage-4" row).
      - 1.7B model trained on ~11T tokens via multi-stage manual rebalancing.
      - Stage 1 (0-6T): 90% English web (60% FineWeb-Edu + 40% DCLM),
        10% StarCoderData (code); no math.
      - Stage 2 (6-8T): 75% web, 20% code, 5% math (OpenWebMath).
      - Stage 3 (8-10T): web + Stack-Edu code + ~10% math.
      - Stage 4 / decay (10-11T, the annealing phase used in the table):
        58% web, 24% code (Stack-Edu), 14% math (FineMath4+/InfiWebMath-3+),
        4% Cosmopedia-v2 (synthetic).
      - The curriculum principle: "Upsampling high-quality math and code during
        the annealing phase, reserving datasets ... for the final stages to
        maximize their impact" (Sec 4 intro) -- exactly the alpha(t) schedule
        modeled in Section D.

[6] https://huggingface.co/blog/smollm
    Ben Allal, Lozhkov, Bakouch et al. (2024-07-16).
    "SmolLM - blazingly fast and remarkably powerful." (Hugging Face blog.)
    Verifies: the SmolLM (v1) corpus composition + the heuristic-mix starting point.
      - SmolLM-Corpus = Cosmopedia-v2 (28B tokens, synthetic textbooks/stories),
        Python-Edu (4B tokens, educational code), FineWeb-Edu deduplicated
        (220B tokens, educational web).
      - 1.7B model trained on 1T tokens; 135M/360M on 600B tokens.
      - FineWeb-Edu: 1.3T tokens filtered by an educational-quality classifier
        trained on Llama3-70B-Instruct annotations -- the classifier-based
        quality-filtering paradigm that makes "tuned mixes" meaningful.
      - Python-Edu converges "3x faster" than unfiltered Python code -- direct
        evidence that data QUALITY (not just quantity) drives small-model gains.

[7] https://huggingface.co/datasets/HuggingFaceFW/fineweb-edu
    Penedo et al. (2024). "FineWeb-Edu." (Hugging Face dataset card.)
    Verifies:
      - FineWeb-Edu = 1.3T tokens of educational web content filtered from
        FineWeb (CommonCrawl) by a classifier -- the dominant "natural web"
        source in modern SLM mixes (SmolLM, SmolLM2, etc.).
      - Establishes that the "web" domain in modern SLM mixing is itself a
        CURATED, classifier-filtered subset, not raw CommonCrawl.

[8] https://arxiv.org/abs/2112.11446
    Rae et al. (2021). "Scaling Language Models: Methods, Analysis & Insights
    from the Gopher." (official, DeepMind.)
    Verifies: the MassiveText mixture (the Gopher/Chinchilla corpus; Section D
    "Chinchilla" row):
      - MassiveText = MassiveWeb + Books + C4 + News + GitHub + Wikipedia.
      - The published domain weights (approx): MassiveWeb ~48%, Books ~27%,
        C4 ~10%, News ~10%, GitHub ~3%, Wikipedia ~2% (sums to 100%).
      - Books corpus ~560B tokens; GitHub ~506B tokens. Establishes the
        "fixed heuristic mix by domain size" regime that DoReMi later
        automated.

[9] https://stanford-cs324.github.io/winter2022/lectures/data/
    Stanford CS324 (2022). "Data" lecture, Foundation Models course.
    Verifies:
      - The composition of GPT-3's dataset: CommonCrawl is "82% of the dataset,
        but contributes only 60%" during training -- the canonical example of
        DOWNSAMPLING a dominant domain (weighted sampling, Section A).
      - The Pile's 22-source mix; C4's classifier filtering; the principle that
        "training on all of [CommonCrawl] doesn't work well" -> curation/mixing
        is mandatory. Independent confirmation of the bundle's thesis.

[10] https://github.com/swyxio/ai-notes/blob/main/Resources/DATASETS.md
     swyxio ai-notes (community-maintained dataset reference).
     Verifies: the Llama 1 pretraining percentages independently of the

NOTES ON FACT-CHECKING / CORRECTIONS vs the original brief
----------------------------------------------------------
The build brief suggested a SmolLM2 mix of "~48% FineWeb-Edu, 25% Python/build,
27% Cosmopedia". The primary SmolLM2 paper (arXiv:2502.02737, source [5]) does
NOT report those numbers; they are characteristic of the earlier SmolLM v1
(source [6]). The actual SmolLM2 recipe is a MULTI-STAGE mix; this bundle uses
the VERIFIED stage-4 annealing shares (58% web / 24% code / 14% math / 4%
Cosmopedia-v2, source [5] Sec 4.5) rather than the brief's single-number
estimate. No unverified fact was shipped.
