Killer gotchas (verbatim from hashmap.py Section E):
  1. Swap-and-pop order: set idx_map[last]=idx before del idx_map[val].
  2. Zero counts ≠ missing key — del state[c] at 0 for clean equality checks.
  3. Use squared distance for geometry: exact integer hash key, no sqrt float error.
  4. Tuples hash, lists don't. Convert [xi,yi] → (xi,yi) before using as a key.
  5. Worst case is O(n) if all keys collide into one bucket; Python randomizes string hashing to prevent this.
  6. TinyURL: md5[:6] can collide; counter + base62 is bijective & collision-free.
