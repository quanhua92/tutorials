raw URLSearchParams → typed, validated, defaulted

A URL search string like ?page=2&sort=desc is raw text — URLSearchParams hands you {{page:"2", sort:"desc"}}: everything is a string, nothing is checked, nothing has a default. TanStack Router’s validateSearch replaces that with a schema (Zod / Valibot / ArkType). The router runs a pipeline — parse → coerce → validate → default — and the component reads route.useSearch() to get a real {'{ page: number, sort: "asc"|"desc", filter?: string }'}. Invalid URLs are rejected (or recovered with fallbacks), never silently passed through.
