Killer gotchas (verbatim from dynamic_programming.py Section E):
  1. State definition is the hardest part — write one English sentence "dp[i] = ..." before any code.
  2. Sentinel for min-DP: Coin Change inits with amount+1, not 0; -1 if still sentinel.
  3. Wrong base for min vs count: min uses dp[0]=0, rest=INF; count uses dp[0]=1, rest=0.
  4. Interval DP fill order: iterate by length, not by index, so dp[i+1][j-1] exists first.
  5. Off-by-one in string DP: dp[i][j] covers s[:i], so use s[i-1], never s[i].
  6. Rolling array: if dp[i] needs only dp[i-1], dp[i-2], use two scalars — O(1) space.
  7. Unbounded = iterate capacity forward; 0/1 = iterate backwards. Wrong direction = wrong problem.
