Killer gotchas (verbatim from graph.py Section D):
  1. Edge direction: prerequisite [a, b] means “b before a”, i.e. edge b → a (NOT a → b). Flip it and your zero-in-degree seeds are silently wrong.
  2. Cycle check: after Kahn’s, always check len(order) == n. Cycle members never reach in-degree 0, so they silently vanish — len < n IS the cycle signal.
  3. GRAY vs BLACK: a back edge hits a GRAY node (on the current path). A cross/forward edge to a BLACK node is fine — not a cycle. A 2-state visited set can’t tell them apart.
  4. 1-indexed nodes: Town Judge uses 1..n. Allocate n+1, loop range(1, n+1). Off-by-one is the #1 bug.
  5. Judge score is n−1, not n: the judge is trusted by everyone else and trusts nobody, so net = n−1. It can never reach n.
  6. Multiple valid orders: Kahn’s may return any valid topo order. Accept any order respecting every edge; don’t assert a specific permutation.
