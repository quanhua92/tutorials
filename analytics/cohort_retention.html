Cohort Retention — Heatmaps, Retention Curves, Churn — interactive COHORT_RETENTION.md cohort_retention.py

Group users by the month they signed up, then ask what fraction return on day N — the N-day retention R(N). Stack cohorts as rows and offsets (D1/D7/D30/D60/D90) as columns and you get a retention heatmap — a triangle, because recent cohorts haven't lived long enough to have a D90. Everything below is recomputed live in JS from the same simulated dataset as cohort_retention.py (7,990 users, 7 monthly cohorts).  |  how this bundle was built →

① Retention heatmap — cohort × horizon (click a cohort to plot its curve)

off = raw SQL output (censored cells read ~0% — the classic math artifact)   0%  →  ~50%
Down a column (vertical): same lifecycle stage, different cohorts — tests product/acquisition quality over time. Across a row: one cohort's decay curve. A diagonal streak = product regression; a vertical red column = measurement bug or seasonality.

② Retention curve plotter — the survival function + exponential fit

The curve is the survival function S(t). A single exponential R(t)=A·e−k·t must decay to zero, so it under-estimates the loyal tail where the curve plateaus — that bend is the core minority. 2025-01 has the full D0–D90 curve (10 points); later cohorts show only their matured horizons.

③ Cohort comparison — one horizon across all cohorts

Censored cohorts (too young for the horizon) render as a dashed n/a — never include them in the denominator, or headline retention looks like it collapsed.

④ Segmentation — 2025-01 split by channel & platform

Headline retention is a blend. Before diagnosing a drop, decompose it: paid trails organic (acquisition-mix shift), iOS runs a few pp above Android (the platform delta).

⑤ Benchmarks × curve-shape diagnostics (2026 medians)

Curve-shape read: steep D1→D7 drop + flat tail = onboarding problem; gradual linear decay = low engagement frequency; power-law = utility apps (normal); S-curve = free-trial conversion gate.
