Killer gotchas (verbatim from vector_databases.py Section E):
  1. Never mix embedding models in one index — cross-model cosine is meaningless; upgrading re-embeds the whole corpus.
  2. For L2-normalized vectors, cosine == dot product — use the faster dot-product index, identical results.
  3. Post-filter can return < k when the filter is selective (<1%); use pre-filter or Qdrant's filterable HNSW.
  4. Curse of dimensionality: above ~2048 dims, distances converge and ANN quality degrades — use PCA.
  5. HNSW memory scales with both n and M (10M vectors → 60+ GB RAM); quantize to cut it 4–32x.
  6. RRF (hybrid search): score(d) = Σ 1/(k + rank), k=60 — no training, scale-robust, handles missing docs.
