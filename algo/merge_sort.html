The stable divide-and-conquer sort. Cut the array in half, recursively sort each half, then merge the two sorted piles (take the smaller top card; on a tie take left → stable). Always Θ(n log n) — there is no bad input. ② n log n shape — every level does O(n) work, there are log₂n levels

What to do: drag n. Merge sort touches each element once per level (O(n)), across log₂n levels. Contrast with selection sort's n² comparisons — the gap widens fast.
levels = ⌈log₂n⌉; work per level = n; total = n·⌈log₂n⌉. Stability is free (take-left on ties); the cost is O(n) auxiliary space for the merge buffer.

③ Complexity — best = average = worst = Θ(n log n) (no degenerate input)

The input order only changes the comparison constant (~½n to n per level), never the number of levels. Quicksort's worst case is Θ(n²); merge sort's never is.
Where it hides: Python list.sort and Java Arrays.sort(Object[]) are TimSort — merge sort + run detection + insertion sort for short runs. Linked-list sort and external (disk) sort are merge sort too.
