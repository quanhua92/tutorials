========================================================================
  SECTION B: the grammar state = Brzozowski derivative
========================================================================
The state IS the residual grammar after consuming the output so far.
derive(state, c) returns the residual after eating one char `c`;
  Null  = forbidden here (masked);  anything else = accepted.

start state        = (('{' letter+) '}')
  valid_chars      = ['{']
  nullable (accept)= False

  derive(state, '{' ) = (letter+ '}')                      ACCEPTED
  derive(state, 'h' ) = (letter* '}')                      ACCEPTED
  derive(state, 'e' ) = (letter* '}')                      ACCEPTED
  derive(state, 'l' ) = (letter* '}')                      ACCEPTED
  derive(state, 'l' ) = (letter* '}')                      ACCEPTED
  derive(state, 'o' ) = (letter* '}')                      ACCEPTED
  derive(state, '}' ) = (done)                             ACCEPTED

final state        = (done)  (nullable -> output complete & valid)
[check] input '{hello}' drives state to accepting (Eps): True -> OK
[check] derive(root, '9') is Null (9 is out of class): True -> OK

========================================================================
  SECTION D: constrained generation trace (seeded sampler)
========================================================================
At every step the mask is rebuilt, a valid token is picked (seeded
RNG over the valid set), and the grammar state advances. The output
is GUARANTEED to parse against the grammar - free sampling is not.

| step | grammar state (residual)            | picked | output so far |
|------|-------------------------------------|--------|---------------|
| 0    | (('{' letter+) '}')                 | {      |               |
| 1    | (letter+ '}')                       | h      | '{'           |
| 2    | (letter* '}')                       | o      | '{h'          |
| 3    | (letter* '}')                       | o      | '{ho'         |
| 4    | (letter* '}')                       | }      | '{hoo'        |
| 5    | (done)                              | (stop) | '{hoo}'       |

Final output: '{hoo}'   (always matches  '{' letter+ '}')

[check] generated output parses against the grammar (ends accepting): True -> OK
[check] generated output starts with '{' and ends with '}': True -> OK

========================================================================
  SECTION E: a richer grammar (JSON object) + multi-char tokens
========================================================================
GBNF source (a tiny {"name": "..."} object grammar):
    # a tiny JSON-ish object:  {"name": "..."}  with optional whitespace
    root   ::= "{" ws "\"name\"" ws ":" ws string ws "}"
    ws     ::= [ \t\n]*
    string ::= "\"" [^"]* "\""

Parsed rules:
    root    ::= (((((((('{' ws) '"name"') ws) ':') ws) string) ws) '}')
    string  ::= (('"' [^"]*) '"')
    ws      ::= [ \t\n]*

START state valid_chars (note whitespace is allowed first):
    ['{']

multi-char vocab: ['{', '{"name"', '{"', 'name', '"name"', ':', '"hi"', '"', '}', ' ', '9']
  valid   = ['{', '{"name"', '{"']
  invalid = ['name', '"name"', ':', '"hi"', '"', '}', ' ', '9']
  -> '{"name"' and '{"' are valid (the whole token fits the state);
     '{\"name\"' even pre-types the key. '{\"' alone is the minimal

STATE after consuming '{"' :
  valid   = ['name']
  invalid = ['{', '{"name"', '{"', '"name"', ':', '"hi"', '"', '}', ' ', '9']
  -> only 'name' (continuing the key) or the rest of the key is live.
