Running a container is two jobs split by a clean seam: the high-level runtime (containerd) pulls images & assembles rootfs; the low-level runtime (runc) creates namespaces + cgroups and execs the process. The contract between them is one config.json (OCI Runtime Spec). Step through the stack, build the overlay rootfs, and toggle namespaces — everything recomputes live, gold-checked against container_runtime.py.

① The runtime stack — step through `docker run nginx`; runc exits after exec

What to do: drag step. Each layer hands a NARROWER contract down. The low-level layer (runc) receives only config.json + rootfs and then exits — the shim becomes the container's parent so it survives containerd restarting.
—
Key seam: runc is oblivious to images/registries/networks; containerd never creates a namespace (it asks runc). That seam is why Docker, containerd, CRI-O and Podman all share ONE runc.
