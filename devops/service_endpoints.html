Kubernetes Service & Endpoints — interactive SERVICE_ENDPOINTS.md service_endpoints.py

A Service is a stable virtual IP (the ClusterIP) backed by ephemeral pods. The Endpoints controller watches pods matching the label selector and rewrites the endpoint list; kube-proxy turns it into iptables DNAT rules. Slide the request count and watch the load distribute — the page re-runs the same seeded xorshift32 simulation as service_endpoints.py Section B.

① The chain: stable ClusterIP → label selector → live pod IPs

What to look at: a client dials the stable 10.96.0.10. That packet never reaches a "VIP node" — kube-proxy on the originating node rewrites the destination to a pod IP (DNAT) before it leaves. Toggle a pod off to see the Endpoints controller drop it from the list and kube-proxy rewrite the rules.

③ Service types & headless DNS — what address the client actually resolves

What to do: pick a Service type. ClusterIP/NodePort/LoadBalancer all funnel to ONE virtual IP; Headless (clusterIP: None) skips kube-proxy entirely and DNS returns the pod IPs directly — that's how StatefulSets give each replica a stable name.
