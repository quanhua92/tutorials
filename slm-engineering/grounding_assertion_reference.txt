[1] https://arxiv.org/abs/2309.15217
    Es, James, Espinosa-Anke, Schockaert (2023), "Ragas: Automated Evaluation of
    Retrieval Augmented Generation" (official; arXiv:2309.15217).
    Verifies: the REFERENCE-FREE RAG evaluation framework that defines the
    "faithfulness" metric this bundle computes. Faithfulness measures "the factual
    consistency of the generated answer against the given context" and is computed
    by (1) identifying the atomic claims in the answer, then (2) checking how many
    are supported by the retrieved context -- i.e. exactly this bundle's
    extract-then-assert pipeline. RAGAS positions RAG as a "natural language layer
    between a user and textual databases, reducing the risk of hallucinations".
    The bundle's per-claim GROUNDED/UNGROUNDED/CONTRADICTED verdicts are the
    atomic, countable form of the RAGAS faithfulness numerator/denominator.

[2] https://docs.ragas.io/en/v0.2.8/concepts/metrics/available_metrics/faithfulness/
    RAGAS documentation, "Faithfulness" (authoritative metric reference page).
    Verifies: the EXACT faithfulness formula and the "faithful" definition --
      "Faithfulness score = |Number of claims in the response that are supported
       by the given context| / |Total number of claims in the response|"
    and "A generated answer is considered faithful if all its claims can be
    supported by the provided context." Confirms faithfulness measures SUPPORT IN
    THE PROVIDED CONTEXT, not agreement with a reference answer (i.e. it is a
    grounding/attribution metric, not a correctness metric). The bundle's
    GROUNDED count / num_claims is the discrete version of this ratio.

[3] https://arxiv.org/abs/2310.11511
    Asai, Wu, Wang, Sil, Hajishirzi (2023), "Self-RAG: Learning to Retrieve,
    Generate, and Critique through Self-Reflection" (official; arXiv:2310.11511).
    Verifies: the IN-MODEL grounding counterpart to this bundle's post-hoc
    grounding. Self-RAG trains a single LM to "adaptively retrieve passages
    on-demand, and generate and reflect on retrieved passages and its own
    generations using special tokens, called reflection tokens", showing
    "significant gains in improving factuality and citation accuracy". This
    bundle is the model-agnostic, training-free counterpart: instead of teaching
    the model to self-critique, we verify the finished answer against a fact
    sheet after generation. Both implement the same claim-support discipline.

[4] https://arxiv.org/abs/2305.14627  (NOTE: the build brief cited 2212.09257;
    that ID is NOT ALCE -- the verified ALCE paper is 2305.14627, Gao et al.
    Princeton NLP, EMNLP 2023. This correction was web-confirmed.)
    Gao, Yen, Yu, Chen (2023), "Enabling Large Language Models to Generate Text
    with Citations" / the ALCE benchmark (official; arXiv:2305.14627).
    Verifies: the per-STATEMENT support check (citation recall) this bundle's
    per-claim verify implements. ALCE defines a statement's citation recall = 1
    iff it has >=1 citation AND concat(cited passages) ENTAILS the statement,
    using an NLI model (TRUE). The paper states this "is in accordance with the
    attributable to identified sources (AIS) framework (Rashkin et al. 2023):
    phi(concat(C_i), s_i)=1 implies that s_i is true based solely on
    concat(C_i)." ALCE also reports "~50% [of] ChatGPT and GPT-4 baselines are
    not fully supported by the cited passages" on ELI5 -- the hallucination rate
    that motivates post-generation assertion.

[5] https://docs.nvidia.com/nemo/guardrails/configure-guardrails/guardrail-catalog/fact-checking
    NVIDIA NeMo Guardrails Library Developer Guide, "Hallucinations &
    Fact-Checking" (authoritative; the self_check_facts output rail docs).
    Verifies: the OUTPUT-RAIL architecture and the strict/lenient POLICY this
    bundle models. "Fact-checking guardrails help ensure that LLM output is well
    grounded in evidence and reduce so-called hallucinations." The self_check_facts
    rail scores the bot response against $relevant_chunks (the retrieved KB
    evidence) and "returns a score between 0.0 (response is not accurate) and
    1.0 (response is accurate)" -- the continuous form of this bundle's discrete
    verdict. Confirms the two deployment modes: (a) the hallucination rail runs in
    BLOCKING mode (the message is withheld) or (b) WARNING mode (the response is
    sent with a caveat) -- exactly the strict vs lenient policy toggle this bundle
    implements. The hallucination rail (no supporting docs) is the
    SelfCheckGPT-style sampling consistency variant; the fact-check rail
    (with $relevant_chunks) is the RAG-grounding variant this bundle models.

NOTABLE VERIFICATION NOTES:
  * The extract -> normalize -> lookup -> assert -> verdict pipeline is the
    shared skeleton of ALL FIVE sources: RAGAS computes faithfulness by extracting
    claims and checking support ([1],[2]); ALCE does it per-statement with an NLI
    entailment check ([4]); NeMo ships it as a configurable output rail ([5]);
    Self-RAG bakes the same critique into the model via reflection tokens ([3]).
  * "Grounded" / "supported" / "entailed" / "attributable" are synonyms across
    these literatures; this bundle uses GROUNDED for the positive case and
    UNGROUNDED (no backing fact) vs CONTRADICTED (backing fact exists but
    mismatches) for the two distinct failure modes -- a split that matters for
    policy (contradictions are always worse than mere absence of evidence).
  * Value normalization ("$5.2B" == "5,200,000,000" == "5.2 billion") is a
    bundle implementation detail, not a claim from the literature; it is the
    obvious pre-step that makes a string-equality lookup behave like a numeric
    comparison. The literature uses an LLM or NLI model for the match; this
    toy uses regex + float-equality so every number prints and the example is
    fully deterministic with zero model dependency.
