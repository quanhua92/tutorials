An immutable, content-addressed object store: hash the body with SHA-256 to get the object key, mint a short URL slug, dedup reposts for free, and serve a 1000× read-heavy workload from the CDN edge. Type a paste to mint a URL, watch the dedup detector collapse identical content to one URL, and race the expiration timer — every hash below is recomputed in pure-JS SHA-256, gold-checked.

① Paste creator — type text, get a hash-based URL

What to do: type or paste anything. The body is SHA-256 hashed (pure JS, below) → the 64-hex digest becomes the object key → an 11-char base62 slug becomes the URL. Same content always yields the same URL (pastebin.py Section 1).

② Dedup detector — identical content ⇒ identical URL

What to do: the same text in both boxes hashes to the same object key, so the second paste is never stored — it just reuses the first URL. Different text ⇒ different keys ⇒ stored separately. This is how content addressing captured 17.44% dedup in pastebin.py Section 2.

③ Expiration countdown — soft-delete hits 404 at expires_at

What to do: pick a TTL, then create. The read path checks expires_at and returns 404 the instant it lapses (soft delete); a nightly job reclaims the bytes. Use a demo bucket + fast-forward to watch it expire instantly (pastebin.py Section 3).
