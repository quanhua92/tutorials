Binary Search — interactive BINARY_SEARCH.md binary_search.py

A sorted array lets you discard half the search space per probe — that is the whole algorithm. Step through the live window [lo..hi], watch the probe (the middle) get compared, and see one half get thrown away each step. The numbers are recomputed live in JS on the identical array as binary_search.py.  |  Interpolation Search estimates the probe →

① The core loop — compare middle, discard half (live window [lo..hi])

What to do: pick a key and hit play. The blue cells are the live window; the gold cell is the probe (mid = lo + (hi-lo)//2); dimmed cells have been discarded. Each step halves the window.
speed 0.7s
live window probe (mid) found / inserted discarded
Worst-case probes for n=16 is floor(log2(16))+1 = 5. A hit usually takes fewer; an absent key takes up to 5 and ends with lo > hi (empty window).

② Lower / upper bound — the insertion points (array with duplicates)

What this shows: lower_bound = first index with arr[i] ≥ key; upper_bound = first index with arr[i] > key. The half-open range between them is the run of values equal to key.
bound (insert point) equal to key
For key=4 on [1,2,2,2,3,3,4,4,4,4,5,6,7,8,8,9]: lower_bound=6, upper_bound=10, so the four 4s occupy [6,10). Matches binary_search.py Section C.
