Killer gotchas (verbatim from trie.py Section D):
  1. is_end is the whole point: search("app") must check it; starts_with("app") must not.
  2. Empty string: search("") = root.is_end; starts_with("") is always true.
  3. Grid DFS — restore the cell: board[r][c]='#' before recursing, restore after. Forgetting it breaks sibling searches.
  4. Grid DFS — prune or TLE: null node.word and pop childless nodes from the parent.
  5. Pass the parent node into the grid DFS so you can del parent.children[ch].
  6. Wildcard "..." can be expensive: k dots → sk branches (trie prunes dead ones).
  7. dict children vs array[26]: dict = flexible + sparse-friendly; array[26] = ~3–5× faster, lowercase only.
