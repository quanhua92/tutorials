Greedy Algorithms — interactive GREEDY.md greedy.py

Greedy = make the locally-best choice at each step, never undo it. It is optimal when an exchange argument holds (activity selection, Huffman, fractional knapsack) and fails otherwise (0/1 knapsack, coin change on [4,3,1]). All numbers are recomputed live in JS from the identical logic as greedy.py.

What to do: hit Step. Greedy sorts by finish time and keeps an activity iff its start ≥ the last finish kept. Watch it pick exactly 4 of 11 activities — the same 4 that the brute-force optimum finds. Each green bar is a kept activity; grey bars are skipped (overlap).

② Fractional knapsack WORKS (240) vs 0/1 greedy FAILS (160 < 220)

Same 3 items, two rules. Fractional lets you take 0.667 of i3 to fill the last 20 units → 240 (optimal). 0/1 greedy-by-density grabs i1+i2, strands 20 units, can't fit i3 → 160. The DP optimum drops i1 for i2+i3 → 220.

③ Huffman coding — greedily merge two smallest (WPL = 224 bits)

Each merge combines the two smallest-frequency nodes. The most frequent char (a, 45) gets the shortest code (0); the rarest (f, 5) gets the longest (1100). Weighted path length = 224, saving 76 bits vs a fixed 3-bit code.
