Killer gotchas (verbatim from cyclic_sort.py Section E):
  1. Out-of-bounds swap: in [0,n] the value n would index nums[n] — always guard correct < n.
  2. Advance i only in the else branch — swap; i+=1 skips the newcomer at i.
  3. Negation mark needs abs() — signs corrupt raw values; index with abs(num)-1.
  4. Duplicate = mailbox already < 0 mid-pass; disappeared = mailbox still > 0 post-pass.
  5. nums[i]==nums[correct] with correct!=i is a duplicate — the guard also stops infinite swap loops.
  6. P287 forbids array modification — use Floyd's cycle detection (value = pointer), not negation.
