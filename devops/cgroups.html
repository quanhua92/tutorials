Linux cgroups — the resource-limiting primitive behind containers CGROUPS.md cgroups.py

A cgroup is a bouncer: it does not change what a process sees (that's namespaces), it changes how much it may take. docker --cpus, --memory, --pids-limit and K8s resources.limits all just write cgroup files. Move the sliders and watch the kernel enforce usage ≤ limit.  |  ← namespaces isolate

① CPU — hard cap (cpu.max) vs relative share (cpu.weight)

What to do: cpu.max is a HARD ceiling (the cgroup is paused above it). cpu.weight only matters when cgroups CONTENTION for busy CPU. Set A's cap and watch it bind; turn caps off and watch weight split the spare. Toggle "max" for unlimited.

② memory.max (OOM kill)  ·  pids.max (fork refused)

memory: above memory.max the kernel cannot "pause" allocation — it kills a process (OOM). pids: above pids.max a fork() returns EAGAIN (refused, not killed) — the fork-bomb fuse.
