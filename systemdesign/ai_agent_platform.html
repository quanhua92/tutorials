An operating system for LLM workflows: the LLM is the CPU, tools are syscalls, memory is RAM+disk, the orchestrator is the scheduler, guardrails are the permission checks, and the token budget is the fair-share quota. Route intents, watch a context window evict vs compact, run a planner→executor→reviewer loop, and meter cost. Every number matches ai_agent_platform.py exactly.

① Tool routing simulator — intent → tool via keyword + cosine

What to do: pick an intent (or type your own). Each tool is scored 0.6·keyword + 0.4·cosine. Green bar wins. Keyword alone is brittle; cosine alone is gamed by synonyms — the fusion routes all 6 intents correctly (ai_agent_platform.py Section 1).

② Context window — FIFO eviction vs compaction + vector recall

What to do: drag the token budget. FIFO drops the oldest non-system message (amnesia); compaction summarizes the oldest 3 into one (recall, at the cost of a summary LLM call). Struck-through = evicted (ai_agent_platform.py Section 2).

③ Orchestration — planner → executor → reviewer (maker-checker)

What to do: switch tasks. The planner emits steps, the executor runs them, the reviewer scores 3 criteria (correctness 0.6 + completeness 0.2 + safety 0.2). Score < 0.8 → REJECT → revise (ai_agent_platform.py Section 3).

What to do: type into either box. INPUT flags PII (SSN/email/phone/card) + prompt-injection. OUTPUT flags safety denylist + length > 200 chars. Validate every boundary, not just the final answer (ai_agent_platform.py Section 4).
