The interface (RFC §9.1, verbatim)

Three methods. The editor (§8 preview) and the export pipeline (§10) BOTH program to this interface — they never reach around it.
interface SlideRenderer:
    load(slide_html, fields, assets)   # mount a slide composition
    render_at(timecode) -> frame       # render at time t (preview or capture)
    duration() -> seconds

§9.4 — one interface, one engine per stage (drift is impossible)

Preview and export never pick different engines. At every stage they share ONE backend, so their frames match. The MIXED case is exactly what the architecture forbids.

The editor and exporter never name an engine — they name the interface. There is no knob that picks "preview engine" vs "export engine" separately.
