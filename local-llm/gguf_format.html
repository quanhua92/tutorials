GGUF Format — interactive hex viewer 📖 GGUF_FORMAT.md 📄 gguf_format.py

A tiny valid GGUF file built byte-by-byte in JavaScript (identical logic to gguf_format.py Section D). Click any byte — or any row in the structure tree — to see which field it belongs to and how it decodes. The four sections: header (magic/version/counts) → metadata KV (typed key/value) → tensor info (name/dims/type/offset) → padding → tensor data (the aligned weight blob). The gold-check pins magic to the exact .py output.
📖 Read the full guide — the GGML→GGUF lineage, binary-layout mermaid, killer pitfalls (offset vs file offset, little-endian, alignment), and the cheat sheet. This page is the interactive companion.

③ Layout formula — how the offsets add up

What this shows: the arithmetic that places tensor_data. Identical to align_up(pos, 32) in gguf_format.py.
The padding exists so tensor_data starts at a multiple of general.alignment (default 32) — that is what lets the OS mmap the weight blob directly. See MMAP_WEIGHTS.md.
