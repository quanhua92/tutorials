📖 Full guide in the .md file  ·  📄 Ground-truth source code in the .py file.
An array is a bookshelf (touching cells, jump straight to slot i); a linked list is a treasure hunt (scattered nodes, follow pointers). Big-O calls the list's middle insert O(1), but the cache turns each pointer hop into a ~100-cycle miss. Step through access, insert, and the cache sim below — all numbers are recomputed live in JS from the identical model as array_vs_linkedlist.py.

④ Cache simulation — read all N elements; array hits L1, list misses every time

What this proves: an L1 line = 64 bytes = 16 ints. Reading array[0] drags array[1..15] into L1 free (spatial locality). A linked-list node's neighbour lives in a different line, so every hop is a fresh miss (pointer chasing). Drag N, then Step through the scan and watch the tally.
