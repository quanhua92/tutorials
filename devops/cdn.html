A CDN is a chain of caches close to users. A request is served from the edge (~10ms), or on a miss falls through to regional, origin shield, and finally the origin (~200ms). Below: the edge map, a cache-hit-rate calculator, cache-key fragmentation, the thundering-herd defense, and a direct-vs-CDN latency comparison. All numbers recompute live from the same model as cdn.py.

① Edge map — user → edge → regional → shield → origin

Each tier serves a fixed fraction of the requests that reach it (the misses from above). Adjust the per-tier hit rates and watch the cumulative hit rate and origin load change.

② Cache hit rate → origin offload → bandwidth cost

What this proves: drag the hit rate. Origin offload tracks it (origin serves only the misses), and the monthly bill collapses because the CDN's price/GB is far below origin egress. The blue check: OK confirms the 99% point matches cdn.py Section E ($207k vs $900k).

③ Cache key — adding a cookie destroys the hit rate

1000 objects, cache capacity 1000. A clean key (host,path) → 100% hit. Adding a high-cardinality field (cookie, User-Agent) explodes the working set past capacity → hit rate collapses.

④ Latency — direct-to-origin vs through the CDN

Per-request outcome is probability-weighted by each tier's hit rate. The CDN's expected latency is the weighted sum; direct-origin pays the full cross-continent round trip every time.

⑤ Origin shield — request coalescing (thundering herd)

A viral URL: 300 edge POPs miss at once. Without coalescing the origin is hammered 300x; with a shield, the first miss fires ONE origin fetch and the rest are served from the shield cache.
