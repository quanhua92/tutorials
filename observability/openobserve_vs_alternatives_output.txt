========================================================================
SECTION A - The field: seven observability storage engines
========================================================================

Seven engines, three economic models. This comparison isolates the
STORAGE + COMPUTE economics of the telemetry tier (logs-heavy), plus
query latency on a shared workload.

The three economic models:

  1. S3-NATIVE (O2, Quickwit, ClickHouse, Loki)
     Data lives as compressed Parquet/chunks on S3. Compute is a handful
     of stateless nodes that do NOT grow with historical data. Storage
     cost ~tracks S3 ($0.023/GB) after compression. THE CHEAPEST AT SCALE.

  2. LOCAL-DISK (Elasticsearch, Splunk)
     Data is tied to EBS/local disk with an inverted/bucket index kept in
     RAM. Both storage (EBS $0.080/GB) and the data-node CLUSTER scale
     with volume. Expensive; Splunk adds a per-GB license on top.

  3. SaaS (Datadog)
     You run an agent only; the vendor owns storage + compute. Priced per
     GB INGESTED (not stored) -> cost is linear in ingest, no ops.

SECTION B - Storage cost math: S3 ($0.023) vs EBS gp3 ($0.080)
========================================================================

The single biggest cost driver is WHERE bytes live. Object storage is
3x cheaper per GB than block storage, BEFORE
compression. S3-native engines win twice: cheaper medium + higher
compression (columnar Parquet/chunks vs an inverted index that ADDS size).

Pricing basis (AWS us-east-1, 2025 public):
  S3 Standard  = $0.023/GB-month
  S3 IA        = $0.0125/GB-month   (cold tier for old data)
  EBS gp3      = $0.08/GB-month  (ES / Splunk hot data)

Storage multiplier = net bytes stored per byte of raw ingest x retention:
  O2         = 1/15  (Parquet + zstd)              -> on S3
  Quickwit   = 1/14  (Parquet + tantivy)           -> on S3
  Loki       = 1/8   (gzip chunks, labels only)    -> on S3
  ClickHouse = 1/10  (MergeTree + zstd)            -> on S3/EBS
  ES         = 2.0x  (primary + replica, net ~raw) -> on EBS
  Splunk     = 1.5x  (bucket index)                -> on EBS
  Datadog    = N/A   (priced per GB ingested)

SECTION C - Compute cost: stateless S3 nodes vs data-node clusters
========================================================================

Storage is only half the bill. The other half is COMPUTE, and here the
two economic models diverge sharply:

  * S3-NATIVE: ingest nodes scale with INGEST RATE (how fast bytes
    arrive). Queriers are stateless and cache Parquet in RAM. Neither
    grows with RETAINED HISTORY -> compute stays ~flat as data ages.

  * LOCAL-DISK (ES/Splunk): data nodes must HOLD the indices on disk +
    heap in RAM. The cluster GROWS with stored volume -> every extra TB
    of retention buys you more nodes, forever.

Same workload for every engine: search 1 TB of retained logs for a rare
error string + GROUP BY service over the last 24h. Latency reflects the
INDEX strategy: Lucene/tantivy posting lists vs Loki's chunk scan vs

  Engine         Index strategy     p50 (ms)  p99 (ms)  vs fastest p50
  -------------  -----------------  --------  --------  --------------
  OpenObserve    Tantivy FTS        109       491       1.4x          
  Elasticsearch  Lucene             78        308       1.0x          
  Loki           labels only        3068      9425      39.5x         
  Quickwit       tantivy            169       695       2.2x          
  ClickHouse     sparse + skip idx  98        433       1.3x          
  Splunk         SPL                255       1196      3.3x          
  Datadog        managed            210       771       2.7x          

Reading the result:

  * Elasticsearch wins needle-in-haystack full-text (Lucene posting list
    = O(1) term lookup). The price you pay: that index is what makes ES
    storage + compute so expensive.

  * Loki LOSES badly on content search: no content index, so
    `|= "error"` must decompress + scan chunks. It is unbeatable on
    COST but you accept slow ad-hoc text search (use labels instead).

  * O2 / Quickwit (tantivy) land close to ES at a fraction of the cost.
    ClickHouse is the aggregation king (columnar vectorised scan).

    starts to bite; Splunk's license dominates.

  * At 1 TB/day: the S3-native model wins by 10-100x. This is O2's home.

WHEN O2 DOES NOT WIN:

  * You need Elasticsearch-grade search relevance ranking / percolation.
  * You need Loki's extreme label-driven cheapness and already run Grafana.
  * You need ClickHouse's analytics SQL depth for non-log workloads.
  * You want zero ops and will pay SaaS per-GB (Datadog/New Relic).
