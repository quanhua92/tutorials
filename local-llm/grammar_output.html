Grammar-Constrained Generation (GBNF) — interactive 📖 GRAMMAR_OUTPUT.md 📄 grammar_output.py

An LLM samples tokens from a probability distribution; GBNF hands the sampler a grammar, and before each token is drawn a mask zeroes every vocab token the grammar would reject. Edit the grammar (or pick a preset), set a vocabulary, then Step through sampling — green tokens are valid, grey tokens are masked. The output is guaranteed to parse. All masks recomputed live in JS with the same derivative engine as grammar_output.py.

② Token mask & step-through sampler

Before each token the mask is rebuilt from the grammar state. green = valid (keeps its model probability), grey = masked (logit set to 0). A token is picked from the valid set; the grammar advances. Output always parses.

③ The grammar state = Brzozowski derivative

The state IS the residual grammar after consuming the output so far. derive(state, c) returns the residual after eating char c: Null = forbidden (masked); anything else = accepted. A token is valid iff derive(state, token) ≠ Null — the whole token fits, not just byte 0.
