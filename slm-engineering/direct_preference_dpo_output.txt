==========================================================================
  SECTION A: the DPO loss (eq.7) on the pinned toy quadruple
==========================================================================
Pinned GOLD inputs: logpi_c=-2.0, logpi_ref_c=-2.5,
                    logpi_l=-3.0, logpi_ref_l=-2.2, beta=0.1

Step 1 -- the implicit reward of each completion (relative to the FROZEN ref):
  r_chosen  = beta*(logpi_c  - logpi_ref_c )
            = 0.1*(-2.0 - (-2.5)) = 0.1*+0.50 = +0.0500
  r_rejected= beta*(logpi_l  - logpi_ref_l )
            = 0.1*(-3.0 - (-2.2)) = 0.1*-0.80 = -0.0800

Step 2 -- the margin (the sigmoid's input):
  z = r_chosen - r_rejected = +0.0500 - (-0.0800) = +0.1300

Step 3 -- the loss:
  sigma(0.13) = 1/(1+e^(-0.13)) = 0.532454
  L = -log sigma(0.13) = -log(0.532454) = 0.6303

GOLD PIN (direct_preference_dpo.html recomputes this identically):
  z = 0.13 ; L = -log sigma(0.13) = log(1+e^-0.13) ~= 0.6303

Same quadruple as Section A. The log-ratio difference
  (logpi_c-logpi_ref_c) - (logpi_l-logpi_ref_l)
  = (-2.0-(-2.5)) - (-3.0-(-2.2)) = +1.30 is FIXED.
beta just SCALES it. So z = beta*0.13 grows linearly with beta.

Reading the table:
  * beta=0.01: margin ~= 0.0013, loss ~= log2. The preference signal
    is so weak the loss barely fires -- the policy is barely nudged.
  * beta=0.1  (the DPO default): margin 0.13, loss 0.6303. The
    signal is meaningful but gentle -- the standard choice.
  * beta=1.0 : margin 1.3, loss ~= 0.24. The signal is strong; the
    policy is pushed hard, but it also drifts far from pi_ref.

==========================================================================
  SECTION D: gradient direction + a micro DPO training loop
==========================================================================
Treat logpi_c, logpi_l as the policy's two trainable log-probs.
logpi_ref_c, logpi_ref_l, beta are FROZEN. Autograd gives the exact
gradient; we take manual SGD steps.

Initial policy log-probs (START at the reference):
  logpi_c = -2.5000  (== logpi_ref_c = -2.5)
  logpi_l = -2.2000  (== logpi_ref_l = -2.2)

Gradient signs (the whole intuition):
  dL/d(logpi_c) = (sigma(z)-1)*beta = -0.050000   <- NEGATIVE
                                                   (so gradient DESCENT
                                                    INCREASES logpi_c)
  dL/d(logpi_l) = (1-sigma(z))*beta = +0.050000   <- POSITIVE
                                                   (so gradient DESCENT
                                                    DECREASES logpi_l)

=> minimizing L pushes chosen UP and rejected DOWN, BOTH relative
   to the frozen reference. The margin grows; the loss falls.

Micro loop: lr=1.0, 5 steps, beta=0.1, ref FROZEN.

==========================================================================
  SECTION E: lineage recap (RLHF 4-model -> DPO 2-model) + shipped recipes
==========================================================================
The alignment stage shed two whole models by solving the RL objective
analytically. Each row removes a failure mode:

| stage         | what it does                            | failure removed                | used by                  |
|---------------|-----------------------------------------|--------------------------------|--------------------------|
| SFT           | supervised fine-tune on demonstrations  | no preference signal yet       | the starting point       |
| RLHF (PPO)    | train reward model, then PPO against it + KL to ref | 4 models in memory; RL unstable; on-policy sampling | InstructGPT, Llama 2-Chat |
| DPO           | closed-form optimal policy -> reward model cancels | no RL; 2 models; one BCE-style loss on pairs | Rafailov 2023; Zephyr-7B-beta |

What shipped models actually used (web-verified; see _reference.txt):

| model           | method                          | models in memory                | beta          | dataset              | source                        |
|-----------------|---------------------------------|---------------------------------|---------------|----------------------|-------------------------------|
| InstructGPT     | RLHF (reward model + PPO)       | 4: policy + ref + reward + value | n/a (KL coef 0.02) | human prefs          | Ouyang 2022 arXiv:2203.02155 |
| Llama 2-Chat    | RLHF (reward model + PPO + safety | 4: policy + ref + reward + value | n/a (KL via reward) | human prefs          | Touvron 2023 arXiv:2307.09288 |
| DPO paper       | DPO (no RL, no reward model)    | 2: policy + ref                | 0.1 (TL;DR, HH); 0.05-0.5 (IMDb) | human prefs          | Rafailov 2023 arXiv:2305.18290 |
| Zephyr-7B-beta  | DPO (distilled, TRL DPOTrainer) | 2: policy + ref                | 0.1           | UltraFeedback (GPT-4 ranked) | Tunstall 2023 arXiv:2310.16944 |

Two patterns:
  * InstructGPT / Llama 2-Chat: the FULL RLHF pipeline -- a separate
    reward model + PPO + a value head + a KL anchor to the ref. Four
    models resident; an RL inner loop; sensitive to tune.
  * DPO / Zephyr: drop the reward model and the RL loop entirely.
    Two models (policy + frozen ref), one classification-style loss.
    Zephyr-7B-beta reaches near-Llama2-Chat-70B chat quality with a
    single 7B base (Mistral-7B) + DPO on UltraFeedback at beta=0.1.
