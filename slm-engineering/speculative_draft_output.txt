==========================================================================
  SECTION C: accept/resample loop (greedy variant -- the GOLD anchor)
==========================================================================
VARIANT: greedy / argmax verification. Walk left-to-right over the GAMMA
draft tokens; ACCEPT while target argmax == draft token; at the FIRST
mismatch RESAMPLE that position by taking the target's own greedy pick.
This preserves the GREEDY target output exactly: 'greedy sampling with
speculative decoding matches greedy sampling without it' (vLLM docs).
The stochastic variant (../llm/SPECULATIVE_DECODING.md Section C) instead
uses rejection sampling to preserve the FULL target distribution p.

Per-position accept/reject decision:

| pos | draft t_i | target argmax | decision                | committed so far        |
|-----|-----------|---------------|-------------------------|-------------------------|
| 0   | the       | the           | ACCEPT (the == the)     | ['the']                 |
| 1   | cat       | cat           | ACCEPT (cat == cat)     | ['the', 'cat']          |
| 2   | sat       | sat           | ACCEPT (sat == sat)     | ['the', 'cat', 'sat']   |
| 3   | on        | a             | REJECT -> resample = a  | ['the', 'cat', 'sat', 'a'] |

Accepted-prefix length R = 3  (positions 0..2 accepted)
Resample/bonus token     = a  (from the target)
Committed this round     = [0, 1, 3, 6]  (['the', 'cat', 'sat', 'a'])
  = 3 accepted + 1 resampled/bonus = 4 tokens
Tokens per target forward = 4

==========================================================================
  SECTION D: speedup sweep (tokens/forward + S vs alpha, gamma, kappa)
==========================================================================
GREEDY VARIANT speedup model:
  E[tokens / target forward] = 1 + alpha + ... + alpha^gamma
                            = (1 - alpha^(gamma+1)) / (1 - alpha)
  S (over plain 1-token/step decode)
    = E[tokens/forward] / (1 + gamma/kappa)
  alpha = per-position agreement rate  P[target argmax == draft argmax]
  kappa = draft_speed_advantage        (draft pass costs 1/kappa of target)

For the TOY round (this file): R=3 accepted of gamma=4 -> tokens/forward
= R + 1 = 4. Plain decode needs 4 target passes for 4 tokens; spec needs 1.
=> ~4x fewer target passes THIS round (before draft overhead).

Expected tokens per target forward E[R+1] = (1-alpha^(gamma+1))/(1-alpha):

| alpha | gamma=2 | gamma=4 | gamma=8 | gamma=8 @ alpha=1 |
|-------|---------|---------|---------|-------------------|
| 0.3   | 1.39    | 1.43    | 1.43    | 9.00              |
| 0.5   | 1.75    | 1.94    | 2.00    | 9.00              |
| 0.7   | 2.19    | 2.77    | 3.20    | 9.00              |
| 0.9   | 2.71    | 4.10    | 6.13    | 9.00              |
| 1.0   | 3.00    | 5.00    | 9.00    | 9.00              |

Key: at alpha=1 (perfect agreement) tokens/forward = gamma+1 (every draft
token accepted + the bonus). At alpha=0.5, gamma=4 -> 1.94 tokens/forward
(about 2x the plain rate of 1).

Speedup S = E[tokens/forward] / (1 + gamma/kappa),  kappa=8 (draft 8x cheaper):

| alpha | gamma=2  | gamma=4  | gamma=8  |
|-------|----------|----------|----------|
| 0.3   | 1.11     | 0.95     | 0.71     |
| 0.5   | 1.40     | 1.29     | 1.00     |
| 0.7   | 1.75     | 1.85     | 1.60     |
| 0.9   | 2.17     | 2.73     | 3.06     |

Key: S>1 needs alpha high enough that the accepted tokens outweigh the
gamma cheap draft passes. With kappa=8 and gamma=4, the draft overhead is
1 + 4/8 = 1.5 target-pass-equivs per round, so spec wins once E[tokens] > 1.5.

Published (labelled, not measured here):
  Leviathan et al 2023: 2x-3x on T5-XXL, identical outputs.
  EAGLE (arXiv:2401.15077): 2.7x-3.5x on LLaMA2-Chat 70B.
  vLLM spec decode: 1.4x-1.6x typical (greedy equality preserved).
