max_flow.py - reference impl. All numbers below feed MAX_FLOW.md.
Ford-Fulkerson method + Edmonds-Karp BFS heuristic (CLRS ch. 26).
Network: CLRS Fig 26.1/26.6 - 6 nodes, 9 edges.

========================================================================
  SECTION A: the flow network (source s, sink t, capacities)
========================================================================
Nodes: s, v1, v2, v3, v4, t
Source S = s   Sink T = t

Each directed edge has a CAPACITY (the pipe width). Flow starts at 0:

| edge      | capacity | flow | residual forward |
|-----------|----------|------|------------------|
|   s->v1  |       16 |    0 |               16 |
|   s->v2  |       13 |    0 |               13 |
|  v1->v3  |       12 |    0 |               12 |
|  v2->v1  |        4 |    0 |                4 |
|  v2->v4  |       14 |    0 |               14 |
|  v3->v2  |        9 |    0 |                9 |
|  v3->t   |       20 |    0 |               20 |
|  v4->v3  |        7 |    0 |                7 |
|  v4->t   |        4 |    0 |                4 |

Constraints every valid flow must satisfy:
  capacity   : 0 <= f(u,v) <= c(u,v)        (never overflow a pipe)
  skew sym.  : f(u,v) = -f(v,u)             (flow one way = anti-flow)
  conserve   : sum_in(v) = sum_out(v)       for v != S, T

Goal: find the largest total flow S -> T. That value is the MAX FLOW.
Capacity leaving S (an upper bound on any flow):
  sum c(S,*) = 29   (S->v1:16 + S->v2:13)
  sum c(*,T) = 24   (v3->T:20 + v4->T:4)
  loose upper bound = min(29,24) = 24; the real max flow is smaller (it is 23, found below).

========================================================================
  SECTION B+C: Edmonds-Karp - BFS augmenting path, augment, repeat
========================================================================
Ford-Fulkerson METHOD: repeatedly find an S->T path in the RESIDUAL
graph and push the bottleneck. EDMONDS-KARP chooses the path by BFS
(shortest in edges) - that single rule gives O(V*E^2).

Residual update per augmenting step:
   forward  r(u,v) -= b    (less room forward)
   reverse  r(v,u) += b    (more room to UNDO later)

========================================================================
  SECTION E: max-flow = min-cut (the max-flow min-cut theorem)
========================================================================
At termination, BFS from S in the RESIDUAL graph reaches exactly the
S-side of the minimum cut. Edges crossing A->B are SATURATED (flow =
capacity); their capacity sum equals the max flow. This is the
max-flow min-cut THEOREM (Ford-Fulkerson 1956): |f*| = capacity(min cut).

S-side (reachable from s in residual graph): ['s', 'v1', 'v2', 'v4']
T-side (the rest):                                   ['t', 'v3']

Cut edges (forward edges from A to B) - all SATURATED:
| edge      | capacity | flow | (flow == capacity) |
|-----------|----------|------|--------------------|
|  v1->v3  |       12 |   12 | True               |
|  v4->v3  |        7 |    7 | True               |
|  v4->t   |        4 |    4 | True               |

capacity(min cut) = 12 + 7 + 4 = 23
max flow |f*|      = 23
GOLD: |f*| == capacity(min cut)?  True  (23 == 23)
[check] max-flow min-cut theorem holds? OK

Why every cut edge is saturated: if an A->B edge had residual > 0,
BFS would have crossed it and put its head in A - contradiction. So
the cut is 'tight': forward edges full, and no backward flow leaks
back into A (else those B nodes would be reachable too).

========================================================================
  SECTION F: applications (bipartite matching, project selection, image segmentation)
========================================================================
Max-flow is a Swiss-army knife: many problems are SOLVED by building
a flow network and running Edmonds-Karp / Dinic on it.

| problem               | how it maps to max-flow                          |
|-----------------------|--------------------------------------------------|
| bipartite matching    | add super-source to all L-nodes (cap 1), all     |
|                       | R-nodes to super-sink (cap 1); max flow = max    |
|                       | matching. (Konig: = min vertex cover.)           |
| max bipartite matching| same; integer capacities -> integral flow.       |
| disjoint paths        | unit capacities; max flow = max edge-disjoint   |
|                       | S->T paths (Menger's theorem).                   |
| project selection     | open-pit mining / selection: max closure in a    |
|                       | dependency DAG -> min cut. (Picard 1976.)        |
| image segmentation    | foreground/background: pixels to S (fg) or T     |
|                       | (bg); capacities from pixel similarity; min cut  |
|                       | = optimal boundary. (Boykov-Kolmogorov 2004.)   |
| baseball elimination  | can team X still win? build a flow of remaining  |
|                       | games; if max flow saturates all game nodes, X   |
|                       | is still alive. (Schwartz 1966.)                 |
| circulation / demands | add edge T->S with cap = infinity; lower bounds |
|                       | via node demands. Feasibility via a super-source.|

The unifying trick: model 'choices' as cuts. The min cut picks the
cheapest consistent partition; the max flow proves it is optimal via
the max-flow min-cut theorem.
