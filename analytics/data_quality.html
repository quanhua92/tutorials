Data Quality — interactive DATA_QUALITY.md data_quality.py

A defense-in-depth view of data quality: validation rules (null / range / format / referential), freshness SLAs tiered by business criticality, anomaly detection (z-score vs IQR vs EWMA), completeness scoring, and a rolled-up quality scorecard. Edit the scanner data and the anomaly series — every check, fence, residual, and score recomputes live from the identical logic as data_quality.py, gold-checked at the bottom.

① Validation scanner — edit sample data, watch each rule fire

What to do: edit the CSV below (or the rule thresholds). The scanner runs four contract checks — null, range, format, whitelist — and labels each PASS / WARN / FAIL. Defaults mirror a slice of data_quality.py Section 1.

② Anomaly detector — z-score vs IQR on an editable volume series

What to do: edit the daily counts (day 13 = 400 drop, day 14 = 5000 spike). Switch method: z-full misses the drop (outliers inflate mean+std), z-baseline (train on first 12 days) catches both, IQR catches both in one robust pass — the same contrast as data_quality.py Sections 3–4.

③ EWMA null-rate monitor — adaptive baseline + spike alert

Daily null rate for users.email. The EWMA baseline lags the series; an alert fires when the one-sided residual beats k·σ. The day-8 spike (2% → 60%) is the SDK-bug anomaly (data_quality.py Section 5).

④ Freshness SLA — tiered lag vs the SLA marker

Event-time lag = NOW − MAX(event_ts). Bar fills to lag; the gold tick is the SLA. Tier 0 breach → page; Tier 1 → Slack; Tier 2 → log only (data_quality.py Section 2).

⑤ Completeness — cell-level non-null rate per column

Necessary but not sufficient: a column can be 100% non-null and still full of wrong values (data_quality.py Section 6).

⑥ Quality scorecard — four dimensions rolled into one score

score = 100 × (pass + 0.5·warn) / total per dimension; WARN counts half. The gauge averages the four (data_quality.py Section 7).
