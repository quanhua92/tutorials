① Eviction visualizer — same stream, three policies, who survives?

LRU re-promotes re-read keys (good for temporal loops) but a big scan evicts the hot set. LFU counts accesses, so hot keys survive scans but new keys start cold. FIFO ignores access entirely. green=live, orange=just inserted, red strikethrough=just evicted.

② Write strategies — latency vs consistency vs safety

The data-loss window = flush interval. At 1000 wps a 10-sec window means 10,000 writes are unflushed if the node dies.

③ Stampede, PER, and sizing — the economics

1000 simultaneous misses on one expiring key — DB reads under each fix
PER refresh probability p = exp(-beta * time_remaining / TTL)
Each request, near TTL expiry, independently refreshes with prob p. The first refresher warms the key; later requests HIT it. No lock, scales horizontally.
