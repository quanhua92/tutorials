📖 Read the full guide — the fixed-mix → PPL-driven → DoReMi lineage, curriculum tables, killer gotchas, cheat sheet. This page is the interactive companion.
The smoothie problem: a corpus is several domains blended by a weight vector α with Σα = 1. Drag the mix sliders, watch the weighted perplexity bar react. Then drag the curriculum timeline and watch the shares anneal from broad-web to reasoning-heavy. Every number is recomputed in JS from the identical formula in dataset_mixing.py.  |  compare with minhash dedup →

① The mix → weighted perplexity: drag α (web/code/synth) → watch the bar react

How to read: the three sliders set each domain's weight (renormalized so Σα=1). The pie shows the mix; the bar shows the weighted perplexity Σ αᵢ·exp(Lᵢ) for the toy losses L = {web:2.3, code:1.8, synth:2.7}. Push mass onto synth (highest PPL) and the weighted PPL rises — that is the key insight: perplexity is a diagnostic, not the objective. (α on the GOLD anchor is pinned to 0.5/0.3/0.2 below.)
web (PPL=9.97) code (PPL=6.05) synth (PPL=14.88 — hardest) weighted PPL bar

② The curriculum: drag training-fraction t → watch α anneal from broad-web to reasoning

How to read: α(t) = (1−t)·α_early + t·α_late. Early (t=0): 80% web for coverage. Late (t=1): 35% web / 30% code / 35% synth — code + synthetic upsampled for reasoning. This is the SmolLM2 schedule (stage-1 ~90% web → stage-4 anneal 58% web).
α_early = {0.80, 0.10, 0.10} → α_late = {0.35, 0.30, 0.35}

③ What shipped models actually mixed (the fixed-heuristic vs tuned eras)

How to read: Llama/Chinchilla set the mix once by domain size (web 48–82%). SmolLM2's stage-4 anneal pushes specialized (code+math+synth) to 42% — far above its stage-1 ~10%.
