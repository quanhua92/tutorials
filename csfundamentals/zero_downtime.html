Canary Rollout — 1% → 10% → 50% → 100%, deterministic hash cohort

0%
cohort rulefnv1a(flag:user) % 100 < pct
users in canary cohort0 / 10
The same user is always in or always out — no sticky-session bias. Advancing the percentage only ADDS users (monotonic), never removes one already in. A user at bucket 16 enters at the 50% gate; one at bucket 88 never enters until 100%.

Flip the switch off — users drop to v1 instantly, with no deploy, no pod restart. This is why feature flags have the lowest blast radius of any strategy: deploy (ship the bits) and release (turn it on) are different events.
