kto_alignment_reference.txt -- web provenance log for the KTO_ALIGNMENT bundle.
Every formula / value below is verified in >=2 independent sources. Each entry
is one URL + a "Verifies:" line stating the exact claim it supports. Distinct
URLs: 6.

-------------------------------------------------------------------------------
[1] https://arxiv.org/abs/2402.01306
    Ethayarajh, Xu, Muennighoff, Jurafsky, Kiela (2024)
    "KTO: Model Alignment as Prospect Theoretic Optimization" (ICML 2024,
    official).
    Verifies: the KTO loss in full. Section 4.1 (Eq 8 + the value-function box)
    defines  L_KTO = E_{x,y}[ lambda_y - v(x,y) ]  with
        r_theta(x,y)   = log pi_theta(y|x) - log pi_ref(y|x)        (no beta here)
        z0             = KL(pi_theta(y'|x) || pi_ref(y'|x))         (reference pt)
        v(x,y)|des     = lambda_D * sigma( beta*(r_theta - z0) )
        v(x,y)|undes   = lambda_U * sigma( beta*(z0 - r_theta) )
    and the biased KL estimator
        z_hat_0 = max(0, (1/m) sum_i log pi_theta(y_{(i+1)%m}|x_i)
                                 - log pi_ref(y_{(i+1)%m}|x_i)).
    Also verifies: beta in [0.01,1.0] ("lower [0.01,0.10] for large SFT'd models;
    higher [0.10,1.00] for small models w/o SFT"); lambda_D=lambda_U=1 default;
    the class-balance rule lambda_D*n_D/(lambda_U*n_U) in [1, 1.5] (Eq 9); and
    the claim that KTO matches/beats DPO at 1B-30B with up to 90% fewer desirable
    examples. Section 3.1 (Eq 4) gives the canonical KT value function
    v(z) = z^alpha (gain) / -lambda*(z0-z)^alpha (loss) with alpha=0.88,
    lambda=2.25 (Tversky & Kahneman 1992 medians).

[2] https://arxiv.org/html/2402.01306v4
    Ethayarajh et al (2024), full HTML rendering of the KTO paper (v4, 19 Nov
    2024) -- the readable text of source [1].
    Verifies: the EXACT branch formulas in typeset form (the value-function box
    immediately under Eq 8), the loss-aversion hyperparameter discussion
    (Section 4.2 "Loss Aversion": lambda_D, lambda_U default 1; the [1, 3/2]
    interval in Eq 9), the recommended Table 1 hyperparameters (Llama-3 8B
    SFT+KTO beta=0.05; Qwen2.5 3B KTO beta=0.50), and the KL-estimate
    paragraph ("we take a biased but convenient estimate by shifting outputs
    in the same microbatch ... z_hat_0 = max(0, ...)"). Second independent
    rendering of every formula in [1].

[3] https://raw.githubusercontent.com/huggingface/trl/main/trl/trainer/kto_trainer.py
    HuggingFace TRL -- the reference KTO trainer implementation (the canonical
    library code that ships KTO to practitioners).
    Verifies: the loss is computed EXACTLY as the paper states. In
    KTOTrainer.compute_loss (lines ~1521-1552):
        chosen_logratios  = chosen_logps  - ref_chosen_logps      # == r_theta
        rejected_logratios= rejected_logps- ref_rejected_logps    # == r_theta
        chosen_losses   = 1 - sigmoid(beta*(chosen_logratios - kl))    # des
        rejected_losses = 1 - sigmoid(beta*(kl - rejected_logratios))  # undes
        loss = concat(desirable_weight*chosen_losses,
                      undesirable_weight*rejected_losses).mean()
    confirming (a) beta is OUTSIDE r_theta and INSIDE the sigmoid, (b) the
    undesirable branch's argument is (z0 - r_theta) -- the sign flip, (c)
    desirable_weight/undesirable_weight == lambda_D/lambda_U (default 1), and
    (d) chosen_rewards = beta*chosen_logratios.detach() is the DPO-style
    "relative reward" form. The DataCollator builds KL_input_ids by cycling
    completions within the batch (y_{(i+1)%m}) -- the mismatched-pair estimator.

[4] https://arxiv.org/abs/2305.18290
    Rafailov, Sharma, Mitchell, Manning, Ermon (2023) "Direct Preference
    Optimization: Your Language Model is Secretly a Reward Model" (NeurIPS 2023).
    Verifies: the SHARED building block r_pi = beta*(log pi_theta(y|x) -
    log pi_ref(y|x)) -- DPO Eq 3 uses it for BOTH chosen and rejected (as a
    difference), KTO uses it per-example. Confirms the lineage: DPO needs
    PAIRED (y_w, y_l) data; KTO relaxes that to unpaired binary. Also verifies
    that pi_ref is the SFT model (DPO Section 4.1) -- the same reference policy
    KTO consumes (🔗 INSTRUCTION_SFT.md).

[5] https://arxiv.org/abs/2302.01306  (NOTE: this is the PROSPECT THEORY citation
    used by the KTO paper, NOT arXiv -- the prospect-theory paper is from 1992)
    ACTUAL primary: Tversky & Kahneman (1992) "Advances in Prospect Theory:
    Cumulative Representation of Uncertainty", Journal of Risk and Uncertainty
    5(4):297-323. DOI: 10.1007/BF00122574 --
    https://doi.org/10.1007/BF00122574
    Verifies: the prospect-theory value function v(z) = (z-z0)^alpha for gains,
    -lambda*(z0-z)^alpha for losses, with MEDIAN fitted parameters alpha=0.88
    and lambda=2.25 (the loss-aversion coefficient) across their human
    subjects. These are the exact values KTO quotes (Section 3.1) and that
    kto_alignment.py Section A reproduces (|v(-0.5)|/v(0.5) == 2.25). This is
    the utility-theory FOUNDATION the KTO loss is derived from.

[6] https://winniexu.ca/research/kto
    Winnie Xu (KTO co-author) -- project page for "Kahneman-Tversky
    Optimization". Independent non-arXiv summary.
    Verifies (plain-language): "KTO matches or exceeds (state-of-the-art) DPO
    performance without using preference data"; it works on "a binary signal of
    whether an output is desirable"; and the link to prospect theory
    (loss-aversion). Confirms the high-level claim and the data-collection cost
    advantage over DPO's paired preferences.

Summary of facts verified >=2 times:
  * KTO loss form L = E[lambda_y - v] with v = lambda*sigma(beta*(+-r))  -> [1],[2],[3]
  * r_theta = log pi_theta - log pi_ref (beta is INSIDE the sigmoid, not in r)
    -> [1],[2],[3]
  * undesirable branch argument is (z0 - r_theta) -- the sign flip           -> [1],[2],[3]
  * z0 = KL term, estimated by mismatched-pair mean clamped at 0             -> [1],[2],[3]
  * beta in [0.01, 1.0]; lambda_D=lambda_U=1 default                         -> [1],[2],[3]
  * class-balance rule lambda_D*n_D/(lambda_U*n_U) in [1, 1.5] (Eq 9)        -> [1],[2]
  * canonical KT value fn v(z)=z^0.88 / -2.25*(-z)^0.88; lambda=2.25         -> [1],[2],[5]
  * shared r_pi building block with DPO; DPO needs pairs, KTO needs binary   -> [1],[4],[6]
  * KTO >= DPO at 1B-30B; handles up to 90% fewer desirable examples         -> [1],[6]

No unverified facts remain. (Note: the original build brief loosely described
z0 as "mean sigma(r)"; that was an imprecise gloss. The ACTUAL z0 is the
mismatched-pair LOG-RATIO mean clamped at 0 -- a biased KL estimator -- per
[1],[2],[3]. kto_alignment.py implements the paper's exact form, not the gloss.)
