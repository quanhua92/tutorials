Interpolation Search — interactive INTERPOLATION_SEARCH.md interpolation_search.py

Instead of always probing the middle, interpolation search estimates where the key is by linear interpolation between the endpoints. On uniform data it averages O(log log n); on clustered data it collapses to O(n). Step through the estimate and watch it one-shot uniform arrays, then crawl on clustered ones.  |  ← Binary Search always probes the middle

① The probe estimate — pos = lo + (key−arr[lo])·(hi−lo) / (arr[hi]−arr[lo])

What to do: pick an array shape and a key, then step. The gold cell is the estimated probe position — where a straight line through the endpoints predicts the key. Uniform data: it lands on the key. Clustered data: it crawls.

The estimate is exact only when the data lies on a straight line. The clustered outlier drags the line so the estimate for a small key keeps landing at lo, advancing one index per probe → O(n).

② Interpolation vs Binary — probe counts, same key, both arrays

What this proves: run both algorithms on the same key and count probes. Interpolation wins on uniform (often a 1-shot hit) and loses badly on clustered small keys.
On the clustered array with a small key, interpolation crawls (~n probes) while binary stays at ~log n. That is the O(n) worst case in action.
