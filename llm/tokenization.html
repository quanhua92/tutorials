▶ How to use this page (no coding needed): in panel ① keep the sample corpus (or type your own), drag the # merges slider, and click Train BPE — each green row is one "glue" step that fuses two pieces. In panel ③ type newer and click Compare to see the three algorithms chop the same word differently.

① Train BPE on a corpus — watch the vocabulary grow merge by merge

What to do: keep the sample text (or type your own), drag the # merges slider, click ▶ Train BPE. Each green row = one "glue" step: the most common neighboring pair is fused into a new piece and added to the vocabulary. The rank is the order learned — used again in ②.
pre-tokenized by whitespace; tie-break = highest count, then earliest first-seen
Green chips = learned merge tokens. The rank column is the merge order used at encode time (lowest rank merges first).

② Encode a word — merges apply by learned rank (GPT-2 bpe())

What to do: type a word like lowest and click Encode (uses the merges from ①). The readout replays the glue-rules in learned order, one row per step, until no learned pair is left. The orange chips are the final token IDs the model consumes.
uses the merges trained in ① (click Train first)
Each step merges the adjacent pair with the lowest rank still present (all occurrences per pass). Stop when no learned pair remains. ⚠ char-level demo: an unseen char → would be byte-fallback in real GPT-2.

③ BPE vs WordPiece vs SentencePiece — same string, id="compare"

What to do: type a word like newer and click Compare. Same word, three chopping rules side-by-side: BPE replays learned merges; WordPiece grabs the longest dictionary match (with ## for continuations); SentencePiece encodes the whole stream using ▁ for spaces.
