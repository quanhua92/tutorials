📖 Read the full guide — the discrete → UMA → MLX lineage, the lazy-fusion math, the memory-budget tables, killer gotchas, cheat sheet. This page is the interactive companion.
The shared-notebook idea: a discrete GPU keeps its own VRAM notebook in another building — the CPU must courier every weight across PCIe. Apple Silicon gives CPU + GPU the same RAM notebook (zero copy), and MLX fuses a run of elementwise ops into one Metal kernel. Drag the footprint slider and toggle eager vs fused. Every number is recomputed in JS from the identical formula in mlx_metal_edge.py.  |  compare with mobile runtime →

① Unified vs discrete memory: drag the footprint → watch discrete FAIL, UMA copy 0

How to read: on a discrete card you must copy the whole footprint into VRAM (cost ∝ bytes); it is only possible if footprint ≤ VRAM. Under UMA the GPU reads the weights from the one shared pool directly — 0 bytes copied, and there is no VRAM ceiling (just system RAM). The default footprint = 8.0 GB (4B params @ fp16) on 6 GB VRAM / 16 GB unified — the Section A gold.
VRAM = 6 GB (discrete)  |  unified RAM = 16 GB (Apple Silicon)
system RAM / unified pool VRAM island (discrete) copy arrow (cudaMemcpy) zero-copy (shared)

② Lazy fusion: toggle eager vs MLX-fused → kernel + allocation bars (3 → 1)

How to read: the toy lazy graph is x → MatMul(W) → Add(bias) → ReLU → out (3 ops). Eager dispatches 3 kernels and writes 3 intermediate arrays to RAM. MLX fuses the bias-add + ReLU (elementwise) into the MatMul epilogue: 1 kernel, 1 allocation — the Section B gold the check badge asserts.
graph: MatMul → Add(bias) → ReLU  (the three fuse)
kernel launches intermediate allocations (RAM writes) fused (intermediates in registers)

③ Memory budget: SLM sizes vs unified RAM (no VRAM) vs 24 GB discrete

How to read: Q4 = 0.5625 B/param (group=32). Under UMA there is no VRAM ceiling — a model fits any unified tier ≥ its footprint. Under discrete it must fit the 24 GB VRAM island to use the GPU.
