① Column-parallel vs Row-parallel — which axis you cut decides the comm cost

What to do: drag TP and watch the matrix W[out,in] get sliced. Column-parallel splits the OUTPUT dim (rows of W) — each rank holds [out/TP, in] and produces a SHARD of the output (no comm in forward). Row-parallel splits the INPUT dim (cols) — each rank holds [out, in/TP] and produces a PARTIAL SUM needing ONE AllReduce.

② The Megatron trick — AllReduce CANCELS in the middle (worked example, recomputed live)

What this proves: for Y = GeLU(X @ A) @ B, TP=2 with A column-parallel and B row-parallel produces Y_0 + Y_1 that bit-matches the full non-TP Y. The AllReduce that would sit between A and B is cancelled because GeLU is element-wise. The green check: OK below confirms the JS recompute matches tensor_parallel.py Section C exactly.

③ Per-layer AllReduce count — exactly 2 per layer (1 attn + 1 MLP)

What this shows: a transformer block under Megatron TP pays ONE AllReduce at the end of the attention sub-block (O proj) and ONE at the end of the MLP sub-block (down proj). Everything between the column-parallel projections and the row-parallel projections is LOCAL (no comm). Compare to DDP, which AllReduces once per OPTIMIZER step (amortized).
