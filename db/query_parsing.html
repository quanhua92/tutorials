Type a SELECT query and watch it flow through the three desks: the lexer slices it into typed tokens, the parser checks the grammar and builds an AST, and the planner reorganizes that into a bottom-up logical plan tree. The pipeline is re-run in JS on every keystroke with the identical rules as query_parsing.py. The green check: OK below confirms the JS round-trip matches the .py gold scalars.
