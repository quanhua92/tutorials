Skip List (probabilistic balanced structure) — interactive 📖 SKIP_LIST.md 📄 skip_list.py

📖 Full guide  ·  📄 Ground-truth source — every panel below recomputes the same model in JS.
A skip list is a stack of sorted linked lists where each lane is a random ~half of the lane below. Search starts at the top-left, runs right while the neighbor is smaller, and drops down otherwise — visiting ~log₂N nodes. The structure below is rebuilt in JS from the identical seeded coin flips (seed=42) as skip_list.py.

① The multi-level skip list — built from [3,6,7,9,12,17,19,21,25,26]

Level 0 (bottom) holds all elements; each higher lane keeps a random ~half (the promoted nodes that got multiple coin-flip heads). Hover a node to see how many heads it took. 12 got 4 heads (top level 4); 6 got 3.

② Search path — start top-left, step right while smaller, drop down otherwise

What to do: pick a target and hit Step. Each right step is one forward hop; the dashed arrow shows a drop-down. The total hops stays ~log₂N. For all 10 elements the worst case is 4 hops (target 26).

③ Insert with coin-flip promotion — flips decide the new node's top lane

To insert 15, flip a biased coin (p=0.5). Keep flipping while HEADS: the count = the node's top level. Then splice it into lanes 0..level. The flips here are the exact next draws from the seeded RNG after the Section A build (matches skip_list.py Section C).

④ Expected height = log₂N — drag N and compare theory vs empirical

Each lane keeps ~half the previous one, so after k lanes ~N·p^k nodes remain; the list runs out around k = log_{1/p}(N). For p=0.5 that is log₂N. The empirical bars average the highest lane over 500 random builds.
p=0.5

⑤ Skip list vs balanced tree — the trade-off

Read it twice: the skip list trades a probabilistic (not worst-case) bound for simpler code and easy concurrency — which is why Redis ZSET and LevelDB memtables use it.
