# slide_index_json — web provenance log
# Every URL has a "Verifies:" line stating exactly what it confirms.
# STEP 2 of HOW_TO_RESEARCH.md is non-skippable; ≥2 sources per concept.

https://github.com/rany2/edge-tts
  Verifies: edge-tts voice IDs follow the "{lang}-{Name}Neural" pattern and MUST end in "Neural". The README's `--list-voices` sample shows real ids like `af-ZA-AdriNeural`, `ar-EG-SalmaNeural`, `ar-AE-HamdanNeural` — none without the suffix. This confirms AGENTS.md pitfall #1: use `vi-VN-HoaiMyNeural`, NOT `vi-VN-HoaiMy` (and `en-US-AriaNeural` likewise). Also confirms the `--voice` / `--write-media` interface the voiceover pipeline wraps.

https://ffmpeg.org/ffprobe.html
  Verifies: ffprobe "gathers information from multimedia streams and prints it in human- and machine-readable fashion" — i.e. media duration is READ from the actual stream (`-show_entries format=duration`, output in seconds), not a value the user authors. This is the authority for slide index.json `duration` being MEASURED per sentence (ffprobe) then concatenated, per AGENTS.md "Voiceover pipeline" step 2.

https://superuser.com/questions/650291/how-to-get-video-duration-in-seconds
  Verifies: the canonical ffprobe duration probe command `ffprobe -i <file> -show_entries format=duration -v quiet -of csv="p=0"` returns the duration in seconds — corroborating that duration is measured from the rendered stream, matching how voiceover.py measures each TTS sentence before concat.

(in-repo) docs/rfc-0001.md §5.3 + §5.6 + §10
  Verifies: the exact slide index.json schema (`id`, `duration` // measured, optional `transition` override of `transition_default`, `fields`, `assets` "sha256:...", `voiceover` {text, voice}); that `fields` reach HTML via `__FIELD__` string replacement because HF `getVariables()` returns {} in sub-comps (§5.6); and that export step 3 reuses the edge-tts + ffprobe + ffmpeg concat pipeline to set `data-duration` to the measured voiceover length (§10).

(in-repo) docs/AGENTS.md "Layout field types" / "Voiceover pipeline" / pitfall #1
  Verifies: the field types (text/image/voiceover) bound in the Properties panel; the measured-duration pipeline (edge-tts each sentence → ffprobe each → ffmpeg concat → write voiceover_timings.json → auto-update data-duration); and pitfall #1 — the `Neural` suffix requirement for edge-tts voice ids.
