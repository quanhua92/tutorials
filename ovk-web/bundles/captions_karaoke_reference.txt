SOURCE 1 (in-repo primary — the export-target spec)
URL:    docs/AGENTS.md  (https://github.com/heygn/openvideokit → AGENTS.md)
Verifies:
  - "Caption styling — CRITICAL RULES": Never use transform / scale() / font-size /
    text-shadow changes on `.word--active` (they cause visual layout shift "jumping").
  - "The correct caption pattern": highlight uses a GSAP DIRECT color tween —
        tl.to(wordSelector, { color: '#ffea00', duration: 0.15, ease: 'power2.out' }, wordStart);
        tl.to(wordSelector, { color: 'rgba(255,255,255,0.4)', duration: 0.15, ease: 'power2.in' }, wordEnd);
  - Base `.word` style: display:inline-block; font-size:48px; color:rgba(255,255,255,0.4);
    margin:0 .1em; text-shadow:0 4px 20px rgba(0,0,0,.8); transition:color .2s ease.
  - Emphasis `.word--emphasis { color:#4ade80; }` is STATIC (no animation).
  - `.word--active { color:#ffea00; }` — active state is ONLY a color change.
  - "What NOT to do" (verbatim banned CSS on .word--active):
        transform: scale(1.15);   font-size: 56px;   text-shadow: 0 0 30px rgba(255,234,0,0.6);
    and banned GSAP:  tl.to(word, { className: '+=word--active', duration: 0.05 }, start);
  - "Why className toggle is banned": reads computed styles before/after and tweens the diff;
    picks up unintended property changes; duration 0.05 too short to see (instant snap);
    conflicts with the CSS `transition` on the element.
  - Voiceover pipeline → captions.build_captions(): "Split each sentence into words";
    "Estimate per-word timing by char ratio"; "Generate caption HTML (phrase + word spans)";
    "Generate GSAP color tween JS (word-by-word highlight)".
  - "Root index.html markers": <!-- CAPTION_LAYER --> → caption HTML;
    /* CAPTION_CSS */ → caption CSS; // CAPTION_TIMELINE → GSAP word-highlight JS.

SOURCE 3 (web — GSAP, the runtime the RFC mandates)
URL:    https://gsap.com/docs/v3/GSAP/Tween/
Verifies:
  - `gsap.to(target, vars)` / `tl.to(target, vars, position)` is a DIRECT property tween:
    "A Tween is what does all the animation work — think of it like a high-performance
    property setter. You feed in targets ... and any properties you want to animate and
    when its playhead moves to a new position, it figures out what the property values
    should be at that point [and] applies them accordingly."
  - Confirms the CORRECT caption pattern: tweening the CSS `color` property directly
    (e.g. { color: '#ffea00', duration: 0.15, ease: 'power2.out' }) is a direct property
    tween — NOT a className toggle. This is the exact mechanism AGENTS.md mandates.
  - `ease` + `duration` are first-class vars; the third `tl.to()` arg is the timeline
    position (the word's start/end time), confirming the per-word scheduling pattern.

--------------------------------------------------------------------------------
SOURCE 4 (web — MDN, why the CSS tokens are banned)
URL:    https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_transforms/Using_CSS_transforms
Verifies:
  - "By modifying the coordinate space, CSS transforms change the shape and position of
    the affected content" — i.e. `transform: scale(...)` repositions/resizes the rendered
    box, which on a karaoke word causes the visible layout shift / "jumping" AGENTS.md bans.
  - Transforms (and analogously font-size changes, which reflow the line box, and
    text-shadow changes, which alter the painted shadow) are exactly the category of
    properties that trigger reflow/repaint mid-animation — the reason they are banned on
    `.word--active` in favor of a pure `color` tween (color is a non-layout property).

NOTES ON UN-VERIFIABLE / HONESTLY-FLAGGED FACTS
- The exact active-color hex for `neon`, `editorial`, and `eco-green` caption styles is
  NOT pinned by AGENTS.md or RFC §5.2 (only `highlight` → #ffea00 is spec-pinned). The
  runnable therefore labels those three "enum-only / template-defined" rather than
  inventing canonical colors. Do not treat any specific neon/editorial/eco-green hex as
  authoritative.
- The char-ratio timing model ("word i dur = sentenceDur * wordChars/totalChars") is the
  algorithm AGENTS.md names ("Estimate per-word timing by char ratio"); the exact
  normalization (chars excluding spaces) is the natural reading and is what makes the
  ratios sum to 1.0 — an invariant the runnable checks.
