# export_pipeline — web provenance log
# Every URL has a "Verifies:" line stating exactly what it confirms.
# STEP 2 of HOW_TO_RESEARCH.md is non-skippable; ≥2 sources per concept.

https://hyperframes.heygen.com/packages/cli
  Verifies: the HyperFrames `render` command renders a composition to MP4 via
  `npx hyperframes render --output output.mp4` (RFC §10 step 5), renders from
  `index.html` by default, supports `-c compositions/intro.html` for a specific
  sub-composition (the very files the assembler writes into compositions/), and
  offers `--docker` for deterministic output. Also verifies `lint` (the §18 lint
  gate), `inspect`/`snapshot` (frame capture + visual diagnostics), and that the
  CLI is "agent-friendly by default ... Output is plain text suitable for parsing"
  — which is why RFC §10 step 6 parses HF stdout for progress. The same runtime
  powers `preview` ("Visual output matches render exactly"), confirming the
  preview↔export-shared-engine claim (RFC §9.4).

https://github.com/rany2/edge-tts
  Verifies: `edge-tts` is the Python TTS the voiceover step (RFC §10 step 3,
  AGENTS.md "Voiceover pipeline") calls to synthesize each sentence to MP3:
  `edge-tts --text "..." --write-media hello.mp3 --write-subtitles hello.srt`.
  Crucially it ALSO writes word-level SRT subtitles (`--write-subtitles`), which
  is how the pipeline can derive per-word caption timing. Verifies voice IDs are
  of the form `vi-VN-HoaiMyNeural` (the `--list-voices` table is all `…Neural`
  names) — i.e. AGENTS.md pitfall #1 (voice IDs need the `Neural` suffix) and the
  `voice` field in RFC §5.3 slide index.json.

https://ffmpeg.org/ffmpeg.html
  Verifies: the FFmpeg concat + silence-padding mechanics the voiceover step
  uses (AGENTS.md "generate_voiceover_smart"): `anullsrc` generates silent audio
  (the silence padding between/around sentences), the concat protocol/demuxer
  joins the per-sentence MP3s into one `voiceover.mp3`, and `ffmpeg` is the
  encoder invoked by HF render for the final MP4. Also covers Main Concepts
  (http://ffmpeg.org/ffmpeg.html#Main-options) for `-i` input / output flags.

https://ffmpeg.org/ffprobe.html
  Verifies: `ffprobe` (part of the FFmpeg suite) is the tool AGENTS.md step 2 of
  the voiceover pipeline uses to "measure actual duration" of each TTS sentence
  MP3 — those MEASURED durations are what step 3 turns into `data-start` /
  `data-duration` and the root `DUR` (RFC §10 step 3). `-show_entries
  format=duration` returns the media duration in seconds.

# ---- in-repo primary sources (the export-target spec) ----

(in-repo) docs/rfc-0001.md §10 "Export Pipeline"
  Verifies: ALL six steps verbatim — (1) Assemble into HF layout
  (index.html + compositions/slide-N.html copied verbatim + assets/); (2) Stamp
  __SLIDE_ID__ and every __FIELD__ from slide index.json; (3) Voiceover: the
  existing edge-tts + ffprobe + ffmpeg concat + timings pipeline, updating
  data-start/data-duration and the root DUR to the measured voiceover length;
  (4) Captions: build_captions() → caption layer + GSAP word-highlight timeline
  into the root index.html markers; (5) Render: npx hyperframes render
  <workspace> --output scene.mp4; (6) Stream progress: parse HF stdout →
  progress events → UI. And the load-bearing line: "The assembler must emit
  output that obeys the full AGENTS.md contract ... AGENTS.md is the
  export-target spec."

(in-repo) docs/rfc-0001.md §5.4/§5.5 (bare <template>), §5.6 (stamp),
  §16 (GSAP+vanilla, no Tailwind), §18 (lint gate + HF stdout parser risk)
  Verifies: the slide index.html copied verbatim at step 1 is a bare
  <template> (§5.4); zero translation follows from §5.5; __FIELD__ stamping is
  the binding because getVariables() returns {} in sub-comps (§5.6); the
  caption GSAP is direct tweens (§16); §18 lists the risk "HyperFrames stdout
  format changes break the progress parser — pin HF version; integration-test
  the parser in CI."

(in-repo) docs/AGENTS.md
  Verifies: the EXPORT-TARGET SPEC the assembler must obey — "Voiceover
  pipeline" (edge-tts → ffprobe measure → ffmpeg anullsrc silence + concat →
  voiceover.mp3 + voiceover_timings.json; smart timing
  prev_end + gap_between_slides, default 0.8s; "Auto-update data-duration + DUR
  var to match actual voiceover length"); "Caption styling — CRITICAL RULES"
  (direct color tween, no transform/scale/font-size/text-shadow on
  .word--active, no className toggle); "Host div format"
  (`position:absolute;inset:0;z-index:100-idx;class="clip"`); "Root index.html
  markers" (<!-- SLIDES_HERE -->, <!-- CAPTION_LAYER -->, /* CAPTION_CSS */,
  // SCENE_TRANSITIONS, // CAPTION_TIMELINE); "Why bare <template>" and
  "Placeholder convention" (__SLIDE_ID__ + __FIELD__, each field id uppercased).
