Database Schema Design — Normalization, ER & Indexing

Your schema is a lease on a building you can never fully evict. Step a table through each normal form (watch the redundancy vanish), build the ER diagram one relationship at a time, then pick a query and watch the composite index light up its leftmost prefix — or break it.

1. Normalization Stepper — UNF → 1NF → 2NF → 3NF → BCNF

Each normal form kills a specific anomaly: 1NF removes repeating groups, 2NF removes partial-key dependencies, 3NF removes transitive dependencies, BCNF demands every determinant be a key. Click a step to watch the table transform. Mirrors section_1nf()…section_bcnf() in database_schema_design.py.

2. ER Diagram Builder

Nouns become tables; relationships become foreign keys. Pick an entity to inspect its columns, then pick a relationship to see the cardinality, the foreign key, and the ON DELETE rule (CASCADE for owned children, RESTRICT for referenced lookups). Mirrors section_er_modeling().

3. Index Visualizer — leftmost-prefix & the 5% rule

Index from the query mix, not the column list. The composite index (user_id, status, created_at) serves any leftmost prefix — pick a query to see which columns light up green and where the prefix breaks. Then check the ~5% selectivity rule that decides whether the planner even uses an index. Mirrors section_indexing().

4. Gold Check — B-tree Depth for 1 Billion Rows
