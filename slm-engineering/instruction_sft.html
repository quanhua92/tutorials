The single most important SFT rule: wrap turns in ChatML (<|im_start|>role\n{msg}<|im_end|>\n), then set labels = -100 on every non-assistant token so the loss trains the model only on the assistant's content (+ its closing <|im_end|>). ① The ChatML token stream — which tokens carry loss?

How to read: the fixed 2-turn conversation below is tokenised into T=24 toy ids (a real ChatML tokenizer produces the same structure with a 32k-128k vocab). Teal = assistant (loss) — the content + the closing <|im_end|> of each assistant turn. Gray = masked (label -100) — system, user, the <|im_start|>assistant\n header, and separators. Toggle to naive to see the BUG: every token becomes a target, so the model is trained to generate the user's turns too.
4 tokens carry loss (assistant, A)
20 tokens masked to -100
24 total tokens (T)
assistant = loss (label = real id) masked = -100 (system / user / header / pad) naive = ALL tokens are targets (wrong)

② The labels array — completion-only vs naive (recomputed live)

How to read: labels = ids.clone() then labels[role != assistant] = -100. CrossEntropyLoss(ignore_index=-100) skips every -100, so loss is averaged over only the 4 assistant positions. Naive SFT keeps every id (no -100) → 23 targets after the causal shift → the model learns to emit user turns. Same arrays, different mask.
