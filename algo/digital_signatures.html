① RSA — sign with d, verify with e, tamper → fail

What to do: type a message, press sign, then toggle tamper to flip the last character. The signature was made on the ORIGINAL bytes, so any change breaks verification.
primes p, q61, 53  →  n = 3233, phi = 3120 public key (n, e)(3233, 17) — publish private key d2753 (= e⁻¹ mod phi) — keep secret message on verifyhello H(m) mod n— signature sig—
press "sign with private d" to seal the message.
sig = H(m)^d mod n ; verify sig^e mod n == H(m). Only the holder of d can make a signature whose e-th power lands on the hash.

② ECDSA — the same job on an elliptic curve (tiny: y²=x³+2x+2 mod 17)

What this shows: a 256-bit ECDSA key matches a 3072-bit RSA key for security, with ~64-byte signatures vs ~256. Here it runs on a curve small enough to print every point. Toggle tamper to break it.
curvey² = x³ + 2x + 2 (mod 17),   G = (5,1),   order n = 19 private d7  →  public Q = d·G = (0,6) messagecurve H(m) mod n— signature (r, s)—
press "sign with private d".
Sign: r = (k·G).x mod n, s = k⁻¹(H(m)+r·d) mod n. Verify: reconstruct X = H(m)·w·G + r·w·Q, accept iff X.x mod n == r.

③ HMAC — symmetric MAC (shared key, fast, NO non-repudiation)

Both sides share the SAME key — so HMAC proves a key-holder sent it, but either party could have made the tag. tag = H((k⊕opad) ‖ H((k⊕ipad) ‖ m)).
