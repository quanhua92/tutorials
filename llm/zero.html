ZeRO (Zero Redundancy Optimizer) — interactive 📖 ZERO.md 📄 zero.py

📖 Read the full guide — architecture diagrams, decision tables, killer gotchas, cheat sheet. This page is the interactive companion.
For newcomers: DDP (→ DDP) replicates the whole model + optimizer state on every GPU — 20N bytes/GPU, 16N of which is pure redundancy. ZeRO partitions that redundancy away, one category at a time: Stage 1 shards the optimizer states, Stage 2 also the gradients, Stage 3 also the parameters. A 7B model drops from 140 GB to 14 GB per GPU on 8 GPUs. Drag K below and watch the bars shrink. Every number is recomputed live in JS from the identical formula in zero.py (a faithful single-process K-rank simulation; the memory & comms arithmetic is exact).  |  → NCCL collectives   → Tensor Parallel   → Pipeline Parallel (splits layers across nodes; orthogonal axis)

① Per-stage memory: DDP → ZeRO-1 → ZeRO-2 → ZeRO-3 (drag K)

How to use: drag K (number of GPUs). Each bar is the per-GPU memory for that stage. Note DDP never moves (it replicates everything); ZeRO-1/2 shrink toward a floor (the full params+grads stay); ZeRO-3 keeps shrinking as 1/K — it shards EVERYTHING. Toggle the model size.

② How a ZeRO-1 step works: ReduceScatter → shard update → AllGather

How to use: drag phase through the 3 steps of a ZeRO-1 optimizer step (K=4, N=8 params). Phase 0 = identical init. Phase 1 = ncclReduceScatter(grads): each rank ends with the averaged grad for its 2-param shard. Phase 2 = each rank updates its OWN shard via the fp32 master. Phase 3 = ncclAllGather: all ranks hold the full updated params (no drift). This is the same data-flow as DDP's AllReduce (🔗 NCCL identity), just split to partition the optimizer state.

③ Communication cost: Stages 1/2 same as DDP; Stage 3 = 1.5× (the only tax)

How to read: per-device communication volume in units of Ψ (#params). DDP's AllReduce ≡ ReduceScatter + AllGather = 2·Ψ. ZeRO-1/2 replace it with ReduceScatter(grad)+AllGather(param) — still 2·Ψ (DeepSpeed: "same communication volume as data parallelism"). Only ZeRO-3 adds a 2nd AllGather (params gathered for backward too) → 3·Ψ = 1.5× DDP (DeepSpeed: "modest 50% increase"). The loose "3×" counts collectives (3 vs 1); by byte volume it is 1.5×.
