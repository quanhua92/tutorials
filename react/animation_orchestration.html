one element → many elements: the three primitives

framer_motion_core animates a single element with a requestAnimationFrame tween. Real UIs need choreography: a list ripples in one-by-one, a dismissed card fades out before it leaves the DOM, a parent flips a label and every child follows. Framer Motion ships three primitives for this — and each is a plain-React mechanism in disguise:

primitivewhat it doesthe plain-React mechanism (used here)
variants named target objects (hidden, visible, exit) — parent switches the label, children inherit it a React.Context carries the active label + stagger step down the tree
stagger children animate with a sequential delay — a wave, not a flash each child reads its index and sets transitionDelay = index * step
AnimatePresence keeps a removed child mounted long enough to play its exit animation park the id in an "exiting" map, render it with the exit target, splice after N ms

The demo below implements all three from scratch — no motion/react import. Items enter with a stagger, the list label flips hidden → visible on mount, and the Remove button parks the row in an exit state for 300ms before the real unmount.
