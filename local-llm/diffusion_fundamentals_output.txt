==========================================================================
  SECTION A: LINEAR NOISE SCHEDULE - the fixed forward coefficients
==========================================================================
DDPM linear schedule (Ho et al. 2020, arXiv:2006.11239):
  beta_t  = beta_1 + (t-1)/(T-1)*(beta_T - beta_1)   for t = 1..T
  alpha_t = 1 - beta_t
  alpha_bar_t = prod(alpha_1 .. alpha_t)   (signal retained)

T = 1000 steps
beta_1   = 0.000100   beta_T = 0.0200
alpha_1  = 0.999900   alpha_T = 0.980000
beta_500 = 0.010040  alpha_500 = 0.989960

==========================================================================
  SECTION C: REVERSE PROCESS - subtract noise, step by step
==========================================================================
The model predicts the noise:  pred_noise = model(x_t, t)
DDPM reverse step (stochastic, eta=1):
  x_{t-1} = (1/sqrt(alpha_t)) * (x_t - ((1-alpha_t)/sqrt(1-alpha_bar_t))*pred_noise)
            + sigma_t * z        where sigma_t = sqrt(beta_t), z ~ N(0,I)

Key insight: if the model returns the TRUE noise (an oracle), then the
predicted clean image is EXACT:
  x0_pred = (x_t - sqrt(1-alpha_bar_t)*noise) / sqrt(alpha_bar_t)  ==  x_0

x_500         = [+0.030, -0.278, +0.145, +0.618, +0.018, -1.662, +0.403, -0.060]
x0_pred       = [+0.600, -0.400, +0.900, -0.200, +0.500, -0.800, +0.300, +0.700]  (oracle: NOISE was the true eps)
x_0 (truth)   = [+0.60, -0.40, +0.90, -0.20, +0.50, -0.80, +0.30, +0.70]
max|x0_pred - x_0| = 1.67e-16   (exact: forward is invertible)
[check] oracle x0_pred recovers x_0 exactly (err < 1e-12) :  OK  (err=1.67e-16)

One DDPM step: x_500 -> x_499
  SNR(x_500) = 0.08529   SNR(x_499) = 0.08623  (SNR rose)

Why this matters: with a PERFECT model you can skip 980 of 1000 steps.
Real models are imperfect, so fewer steps -> more error -> see Section E.

==========================================================================
  SECTION D: SCHEDULERS - how to take fewer (better) steps
==========================================================================
A scheduler decides WHICH timesteps to visit and the exact reverse
update. Same model, different scheduler = wildly different speed.

Noise schedule comparison (alpha_bar_t = signal kept):
| step t | linear alpha_bar | cosine alpha_bar |
|--------|-----------------|------------------|
|      1 |        0.999900 |         0.999959 |
|    100 |        0.897018 |         0.972093 |
|    250 |        0.524085 |         0.847012 |
|    500 |        0.078587 |         0.493844 |
|    750 |        0.003351 |         0.144272 |
|   1000 |        0.000040 |         0.000000 |
  Cosine keeps far more signal than linear mid-walk (0.8470 vs 0.5241
  at t=250), better for small images. Linear is the canonical default.

==========================================================================
  SECTION F: LATENT DIFFUSION - diffuse in a 48x smaller space
==========================================================================
Stable Diffusion / Flux do NOT diffuse in pixel space. A VAE encoder
compresses pixels -> a small latent, diffusion runs there, a VAE
decoder turns the denoised latent back into pixels.

pixel space  : 512x512x3 = 786,432 values
latent space : 64x64x4 =  16,384 values
compression  : 48.0x less math per diffusion step

Same idea, bigger video (LTX-Video, 720x480x121 frames):
  video pixels =  125,452,800   video latent =   345,600   (363x)
