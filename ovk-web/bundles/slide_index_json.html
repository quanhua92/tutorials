The measured-duration pipeline (the key "why")

voiceover.text → split into sentences
each sentence → edge-tts → temp.mp3
ffprobe each → measure ACTUAL duration (seconds)
ffmpeg concat → assets/voiceover.mp3
sum + gaps → measured slide duration
The user never types duration — the pipeline measures it from the real TTS audio via ffprobe. Export step 3 (RFC §10) reuses this and sets data-duration.

Voice validator — the edge-tts "Neural" suffix

AGENTS.md pitfall #1: voice ids MUST end in Neural. Try vi-VN-HoaiMy (bad) or en-US-AriaNeural (good).
