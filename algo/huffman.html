Huffman coding gives short codes to common symbols and long codes to rare ones, optimally. Watch the tree grow by repeatedly marrying the two lightest nodes, then see encode/decode bit-by-bit. The codes are prefix-free, so decoding is unambiguous.  |  RLE exploits runs of identical symbols →

① Frequency table + merge trace — always marry the two lightest

What to do: type input (or pick a preset). The frequency bars show how skewed the symbols are. Then step the merge trace: each step pops the two lightest heap nodes and merges them into a parent whose weight is their sum. Rare symbols sink deep.
step 0

② The code tree — left edge = 0, right edge = 1 (codes are prefix-free)

The final tree. Each leaf's path from the root (reading 0/1 off each edge) is its codeword. No codeword is a prefix of another, so the decoder walks bit-by-bit and emits a symbol at every leaf — no separators. Hover/click a leaf to trace its path.

③ Encode + decode — bit-exact round-trip

What to do: step the encoder to watch each symbol append its codeword to the bit stream. Then watch the decoder walk the tree one bit at a time, emitting a symbol at each leaf — exact inverse, prefix-free guarantees no ambiguity.
